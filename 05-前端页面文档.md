# 前端页面文档

## 1. 文档说明

### 1.1 文档概述

本文档定义绩效管理系统的前端页面结构与路由配置，作为六端对齐的重要组成部分。所有前端页面开发必须严格遵循本文档定义的页面结构和API调用规范。

### 1.2 六端对齐规范

#### 对齐基准
- **API契约**：docs/04-API接口文档.md + docs/openapi.yaml
- **数据库模型**：backend/database/migrations/001_init.sql

#### 对齐要求
- 所有前端页面必须有对应的API端点
- 请求参数必须与API契约定义的参数100%一致
- 响应数据解析必须严格使用API契约定义的字段
- 表单校验规则必须与API参数校验规则一致

### 1.3 技术规范

#### 前端技术栈
| 层级 | 技术选型 |
|------|----------|
| 框架 | Vue 3 + TypeScript |
| UI组件库 | Element Plus |
| 状态管理 | Pinia |
| 路由 | Vue Router |
| 样式 | Tailwind CSS |
| 构建工具 | Vite |
| HTTP客户端 | Axios |

## 3. 页面组件清单

### 3.1 认证相关页面

#### 3.1.1 登录页面

**页面路径**：`/login`

**页面功能**：用户登录认证

**API调用**：
- `POST /api/auth/login` - 用户登录

**表单字段**：
| 字段名 | 类型 | 必填 | 校验规则 | 对应API字段 |
|--------|------|------|----------|-------------|
| username | string | 是 | 不能为空，长度2-50 | username |
| password | string | 是 | 不能为空，长度6-50 | password |

**响应处理**：
- 成功：保存token到localStorage，跳转到仪表盘
- 失败：显示错误提示

#### 3.1.2 注册页面

**页面路径**：`/register`

**页面功能**：新用户注册

**API调用**：
- `POST /api/auth/register` - 用户注册

**表单字段**：
| 字段名 | 类型 | 必填 | 校验规则 | 对应API字段 |
|--------|------|------|----------|-------------|
| username | string | 是 | 不能为空，长度2-50，唯一 | username |
| password | string | 是 | 不能为空，长度6-50 | password |
| email | string | 否 | 邮箱格式 | email |
| phone | string | 否 | 手机号格式 | phone |

**响应处理**：
- 成功：跳转到登录页面
- 失败：显示错误提示

### 3.2 仪表盘页面

#### 3.2.1 仪表盘

**页面路径**：`/`

**页面功能**：系统首页仪表盘，展示关键数据和统计信息

**API调用**：
- `GET /api/auth/me` - 获取当前用户信息
- `GET /api/users` - 获取用户统计
- `GET /api/assessments` - 获取绩效统计
- `GET /api/goals` - 获取目标统计

**展示内容**：
- 用户统计（总数、今日新增、活跃用户）
- 绩效评估统计（进行中、已完成、平均分）
- 目标完成统计（进行中、已完成、完成率）
- 待办事项（待审批、待处理）

**页面路径**：`/users`

**页面功能**：创建新用户

**API调用**：

- 成功：跳转到用户列表
- 失败：显示错误提示
