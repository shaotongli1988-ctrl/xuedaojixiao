## 结论

## 已冻结资源边界

1. 对外模块资源固定为：
   - `annualInspection`
   - `honor`
   - `publicityMaterial`
   - `designCollab`
   - `expressCollab`
2. 内部允许复用共享 entity / service / 壳层，但内部命名不作为对外契约。
3. 每个模块首批接口固定为：
   - `page / info / stats / add / update / delete`
4. 首批不开放：
   - 真实文件上传 / 下载
   - 快递同步 / 配置 / 测试连接
   - 设计评论流 / 审稿流
   - 文件管理 / 知识库
   - 车辆管理 / 知识产权管理

## 权限冻结结论

1. 页面入口权限固定为：
   - `performance:annualInspection:page`
   - `performance:honor:page`
   - `performance:publicityMaterial:page`
   - `performance:designCollab:page`
   - `performance:expressCollab:page`
2. 动作权限按模块固定为 `performance:<module>:(page|info|stats|add|update|delete)`。
3. `HR 管理员`：主题22 全量。
4. 部门经理：首批无后台入口。
5. 员工：首批无后台入口。

## 状态集合

1. `annualInspection`：`draft / preparing / submitted / approved / rejected / expired`
2. `honor`：`draft / published / archived`
3. `publicityMaterial`：`draft / review / approved / published / offline`
4. `designCollab`：`todo / in_progress / review / done / cancelled`
5. `expressCollab`：`created / in_transit / delivered / exception / returned`

## 数据边界补充规则

1. 所有记录统一返回共享字段与 `extJson` 轻量摘要，不返回真实二进制正文。
2. 内部 `moduleKey` 必须为五个固定枚举之一，不允许写入 `documentCenter / knowledgeBase / vehicle / intellectualProperty`。
3. `extJson` 首批只允许承接页面差异化元数据摘要，不允许存储：
   - 存储凭证 / Token / API Key
   - 快递系统连接配置
   - 设计稿二进制正文
   - 证照附件全文
4. `stats` 只返回当前权限范围内的聚合计数与更新时间，不返回跨类型明细正文。
