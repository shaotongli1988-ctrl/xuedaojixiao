## 范围

1. 实现主题10前端页面：
   - 列表
   - 详情
   - 创建
   - 编辑
2. 前端只对接冻结允许的 `contract page / info / add / update / delete` 五个接口。
3. 页面只对 `HR` 显示入口和动作，经理 / 员工无入口。
4. `draft` 合同可编辑、可删除；`active` 不可删除；`expired / terminated` 不可编辑、不可删除。
5. 列表、详情、表单只展示冻结允许字段，`salary` 仅在 `HR` 页内明文展示。

## 非目标

1. 不改后端 entity / service / controller
2. 不改 `menu.json`
3. 不做 PDF 预览、电子签、签名图片、签署轨迹、审批历史、归档下载
4. 不做菜单权限联调收口
5. 不扩到合同签约执行链路或主题5自动审批流

## 验收口径

1. 页面资源名固定为 `contract`。
2. `HR` 可走通列表 / 详情 / 创建 / 编辑主链。
3. 删除动作只允许对 `draft` 显示并发起。
4. `expired / terminated` 编辑入口关闭，`active` 不提供删除入口。
5. 表单校验至少覆盖：
   - 必填项
   - `endDate > startDate`
   - 枚举值合法
   - 长度 / 数值边界
6. 前端必须通过：
   - `corepack pnpm run type-check`
   - `corepack pnpm run build`
