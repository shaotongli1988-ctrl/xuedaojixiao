## 阻塞解除后至少应覆盖的测试矩阵

### 正常路径

1. `HR` 可分页查看采购增强视图和供应商。
2. `HR` 可完成 `add / update / delete / submitInquiry / submitApproval / approve / reject / receive / close`。
3. 部门经理可在本人部门树范围内完成采购增强主链与流程动作。
4. 部门经理可只读查看供应商摘要与详情。
5. 报表接口对 `HR` 返回全量，对经理返回部门树范围统计。

### 异常路径

1. 员工访问采购增强接口、报表接口和供应商接口被拒绝。
2. 经理访问供应商新增 / 编辑 / 删除接口被拒绝。
3. 非法流转被拒绝：
   - 非 `draft` 不可删除
   - 非 `pendingApproval` 不可 `approve / reject`
   - 非 `approved / received` 不可 `receive`
   - 非 `approved / received` 不可 `close`
4. 非法 `status`、无效 `supplierId / departmentId / requesterId`、重复 `orderNo / code` 被拒绝。

### 边界路径

1. 采购订单分页参数边界、空关键字、部门树边界都必须覆盖。
2. `items / inquiryRecords / approvalLogs / receiptRecords` 的空值、部分值和累计收货边界必须覆盖。
3. `totalAmount`、`orderDate`、`expectedDeliveryDate`、`title`、`name`、`code`、`contact*` 字段长度 / 空值边界必须覆盖。
4. 部门经理跨部门树访问采购订单时必须被裁剪或拒绝。

### 权限与脱敏

1. `HR` 可见完整 `totalAmount / items / inquiryRecords / approvalLogs / receiptRecords / bankAccount / taxNo / contact*`。
2. 部门经理仅可见本人部门树范围内采购增强详情，供应商敏感字段只返回脱敏摘要。
3. 员工无页面、无接口、无数据返回。

### 状态

1. 采购订单：`draft -> inquiring -> pendingApproval -> approved -> received -> closed` 合法。
2. `pendingApproval -> draft` 驳回合法。
3. `draft / inquiring / pendingApproval -> cancelled` 合法。
4. 供应商：`active -> inactive -> active` 合法。
5. 删除约束只允许：
   - `draft` 采购订单删除
   - `inactive` 且无关联有效采购订单的供应商删除
6. 供应商状态变化不自动驱动采购订单状态变化。

## 任务拆分判断

1. 当前后续实现继续保持为：`一个主题三窗口`。
2. 当前不建议拆成“采购订单管理 / 供应商管理”两个子主题，原因是：
   - 供应商仍是采购单增强链路的直接依赖主数据
   - 询价、审批、收货和报表都围绕同一张采购主单
   - `订单管理` 只是采购单执行视角，不是第二资源
3. 只有当后续新增以下任一方向时，才重新评估拆题：
   - 采购增强进一步扩到付款 / 对账 / 库存总账
   - 供应商扩展到资质 / 评级 / 结算中心 / 合同中心
