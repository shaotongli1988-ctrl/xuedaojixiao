## 阻塞解除后至少应覆盖的测试矩阵

### 正常路径

1. `HR` 可分页查看采购订单和供应商。
2. `HR` 可创建、编辑采购订单和供应商。
3. `HR` 可按冻结状态限制删除采购订单和供应商。
4. 部门经理可在本人部门树范围内创建、编辑、查看采购订单。
5. 部门经理可只读查看供应商摘要与详情。

### 异常路径

1. 员工访问采购订单与供应商接口被拒绝。
2. 经理访问供应商新增 / 编辑 / 删除接口被拒绝。
3. 非法删除条件被拒绝：
   - 非 `draft` 采购订单不可删除
   - 非 `inactive` 供应商不可删除
   - 存在有效采购订单引用的供应商不可删除
4. 非法 `status`、无效 `supplierId / departmentId / requesterId`、重复 `orderNo / code` 被拒绝。

### 边界路径

1. 采购订单分页参数边界、空关键字、部门树边界都必须覆盖。
2. `totalAmount`、`orderDate`、`title`、`name`、`code`、`contact*` 字段长度 / 空值边界必须覆盖。
3. 部门经理跨部门树访问采购订单时必须被裁剪或拒绝。

### 权限与脱敏

1. `HR` 可见完整 `totalAmount / bankAccount / taxNo / contact*`。
2. 部门经理仅可见本人部门树范围内采购订单金额，供应商敏感字段只返回脱敏摘要。
3. 员工无页面、无接口、无数据返回。

### 状态

1. 采购订单：`draft -> active -> cancelled` 合法。
2. 供应商：`active -> inactive -> active` 合法。
3. `cancelled` 采购订单不可恢复。
4. 删除约束只允许：
   - `draft` 采购订单删除
   - `inactive` 且无关联有效采购订单的供应商删除
5. 供应商状态变化不自动驱动采购订单状态变化。

## 任务拆分判断

1. 当前后续实现建议保持为：`一个主题三窗口`。
2. 当前不建议拆成“采购订单管理 / 供应商管理”两个子主题，原因是：
   - 供应商是采购订单首批主链的直接依赖主数据
   - 两条主链共享同一组角色、脱敏和部门树范围规则
   - 当前首批范围仍是最小台账链路，独立拆题只会增加跨主题依赖
3. 只有当后续新增以下任一方向时，才重新评估拆成两个子主题：
   - 采购订单扩展到审批 / 收货 / 对账 / 付款
   - 供应商扩展到资质 / 评级 / 结算中心 / 合同中心
