## 资源名

1. 统一资源名固定为：`recruitPlan`

## 最小接口集合

1. `page`
2. `info`
3. `add`
4. `update`
5. `delete`
6. `import`
7. `export`
8. `submit`
9. `close`
10. `void`
11. `reopen`

## 角色权限边界

1. `HR`：
   - 拥有 `page / info / add / update / delete / import / export / submit / close / void / reopen`
2. 部门经理：
   - 拥有 `page / info / add / update / delete / import / submit / close / void / reopen`
   - 不拥有 `export`
   - 仅限本人部门树范围
3. 员工：
   - 无入口
   - 无任何主题16权限

## 状态集合与合法流转

1. 状态集合固定为：
   - `draft`
   - `active`
   - `voided`
   - `closed`
2. 合法流转固定为：
   - 新建保存 -> `draft`
   - 导入成功 -> `draft`
   - `draft` 编辑保存 -> `draft`
   - `draft` 提交备案 -> `active`
   - `draft` 删除 -> 物理删除
   - `active` 关闭 -> `closed`
   - `active` 作废 -> `voided`
   - `closed` 重新开启 -> `active`
   - `voided` 重新开启 -> `active`
3. 非法流转固定为：
   - `draft` 不可直接关闭、作废或重新开启
   - `active` 不可回退 `draft`，也不可编辑或删除
   - `closed` / `voided` 不可编辑、提交、删除、关闭或再次作废

## 数据口径和字段边界

1. 列表 / 详情最小字段固定为：
   - `id`
   - `title`
   - `targetDepartmentId`
   - `targetDepartmentName`
   - `positionName`
   - `headcount`
   - `startDate`
   - `endDate`
   - `recruiterId`
   - `recruiterName`
   - `requirementSummary`
   - `status`
   - `createTime`
   - `updateTime`
2. 不返回：
   - 简历全文、候选人联系方式、附件摘要或附件二进制
   - 面试评语全文、录用决策全文
   - 候选人名单明细
   - 预算明细、薪资金额、Offer 信息
3. 导出字段固定为同一套招聘计划摘要字段，不额外导出联系方式、预算、Offer、审批或跨主题明细。
4. 导入模板固定只允许招聘计划摘要字段，不允许夹带候选人、面试、录用或预算明细对象。
5. 主题16首批允许弱引用或不引用职位标准，但不要求强绑定职位标准。

## 终态规则

1. `active` 不允许再编辑核心正文。
2. `closed` 与 `voided` 只允许执行 `reopen`，不允许其他写动作。
3. `delete` 只允许 `draft` 且无下游引用时执行。
4. 关闭、作废、重新开启都不自动改写主题8、15、18主状态。
