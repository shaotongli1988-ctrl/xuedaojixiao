## 方案选择

### 备选方案

1. 单入口工作台 + 业务页面栈路由。
2. 工作台 / 我的双 Tab 架构。
3. 四业务页并列 Tab 架构。

### 推荐方案

1. 采用“单入口工作台 + 业务页面栈路由”。
2. 保留现有 `pages/user/login` 作为登录入口，保留 `pages/index/home` 作为首批唯一业务首页。
3. 业务页面统一进入 `pages/performance/*` 路由组，详情和动作页使用隐藏二级路由，不新增底部多 Tab 业务导航。

### 选择理由

1. 与冻结范围完全一致，不会额外引入“我的”或多业务并列入口。
2. 经理与员工角色差异大，单入口工作台更容易按权限裁剪卡片和入口。
3. 栈路由更适合“列表 -> 详情 -> 动作页 -> 返回上一层”的移动端操作链路。
4. 现有 `cool-uni` 已具备登录页和首页壳能力，最小改造成本最低。

## 信息架构结论

### 一级结构

```text
登录
└─ 工作台
   ├─ 我的考核
   ├─ 待我审批（仅经理）
   ├─ 目标
   └─ 360 环评
```

### 路由树结论

```text
/pages/user/login                         登录
/pages/index/home                         工作台

/pages/performance/assessment/list        我的考核
/pages/performance/assessment/detail      评估详情（共享二级页）
/pages/performance/assessment/edit        评估草稿编辑（二级动作页）

/pages/performance/approval/list          待我审批

/pages/performance/goal/list              目标
/pages/performance/goal/detail            目标详情
/pages/performance/goal/edit              目标编辑（二级动作页）
/pages/performance/goal/progress          目标进度更新（二级动作页）

/pages/performance/feedback/list          360 环评
/pages/performance/feedback/detail        环评任务详情
/pages/performance/feedback/submit        环评填写（二级动作页）
/pages/performance/feedback/summary       环评汇总（二级查看页）
```

### 路由分层原则

1. `login` 与 `workbench` 属于公共入口层。
2. `list` 页面是首批可见业务页。
3. `detail / edit / progress / submit / summary` 都是隐藏二级路由，只能从首批页或其子页进入。
4. 直达二级路由时，若缺少上级上下文，返回目标固定降级到工作台。

## 页面状态设计

### 全局规则

1. 未登录：
  - 访问受保护页面直接跳登录页。
2. 无权限：
  - 工作台不展示入口卡片。
  - 直达页面时展示无权限占位，并提供“返回工作台”。
3. 接口错误：
  - 顶部错误提示 + 页面级错误占位 + “重试”按钮。
4. 空状态：
  - 使用页面级空占位，不显示空白列表。

### 页面级状态矩阵

| 页面 | 空状态 | 错误态 | 无权限态 |
| --- | --- | --- | --- |
| 登录 | 不适用 | 验证码 / 登录失败提示并允许重试 | 已登录访问时直接跳工作台 |
| 工作台 | 无待办时展示“当前无待处理事项” | 首页摘要请求失败时保留卡片框架并允许重试 | 不适用 |
| 我的考核 | 展示“当前暂无考核记录”，提供返回工作台 | 列表失败时显示重试 | 非员工或无权限时展示无权限占位 |
| 待我审批 | 展示“当前暂无待审批记录” | 列表失败时显示重试 | 非经理或无权限时展示无权限占位 |
| 目标 | 展示“当前暂无目标” | 列表或详情失败时显示重试 | 无权限时展示无权限占位 |
| 360 环评 | 展示“当前暂无环评任务” | 列表或详情失败时显示重试 | 无权限时展示无权限占位 |
| 评估详情 / 目标详情 / 环评详情 | 详情不存在时展示“记录不存在或已失效” | 加载失败时显示重试 | 无权限时展示无权限占位并回来源页 |
| 编辑 / 提交 / 进度更新页 | 不适用 | 提交失败保留表单并提示错误 | 无权限时禁止进入并回详情页 |
| 环评汇总 | 无汇总数据时展示“当前暂无汇总结果” | 加载失败时显示重试 | 无权限时展示无权限占位 |

## 页面进入与返回规则

1. 登录成功固定 `reLaunch` 到工作台，不回到登录页。
2. 工作台进入业务页统一使用 `navigateTo`。
3. 列表进入详情统一使用 `navigateTo`。
4. 二级动作页提交成功后不回工作台，只回对应详情页。
5. 详情页若从列表进入，返回列表；若为深链直达，返回工作台。
6. 无权限拦截统一优先回来源页，来源缺失时回工作台。
