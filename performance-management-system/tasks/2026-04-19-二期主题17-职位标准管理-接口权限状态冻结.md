## 资源名

1. 统一资源名固定为：`jobStandard`

## 最小接口集合

1. `page`
2. `info`
3. `add`
4. `update`
5. `setStatus`

## 状态与启停用

1. 状态集合固定为：
   - `draft`
   - `active`
   - `inactive`
2. 合法流转固定为：
   - 新建保存 -> `draft`
   - `draft` 编辑保存 -> `draft`
   - `draft` 启用 -> `active`
   - `active` 编辑保存 -> `active`
   - `active` 停用 -> `inactive`
   - `inactive` 重新启用 -> `active`
3. 非法流转固定为：
   - 首批无删除
   - 不引入 `archived / retired / reviewing` 等扩展状态

## 引用关系与兼容规则

1. 首批把岗位画像、任职要求和面试评价模板都冻结为摘要字段，不引入独立配置子资源。
2. 首批不要求招聘计划、面试或录用必须引用 `jobStandard`。
3. `inactive` 不自动清空既有引用快照，也不反向改写招聘计划、面试或录用对象。

## 字段结构

1. 最小字段固定为：
   - `id`
   - `positionName`
   - `targetDepartmentId`
   - `targetDepartmentName`
   - `jobLevel`
   - `profileSummary`
   - `requirementSummary`
   - `skillTagList`
   - `interviewTemplateSummary`
   - `status`
   - `createTime`
   - `updateTime`
2. 不返回：
   - 简历全文、候选人联系方式、附件对象
   - 面试评语全文、录用决策全文
   - 课程、证书、能力画像详情对象
   - 薪资区间、预算金额、评价模板设计明细、题库题目或评分规则编辑对象

## 角色权限边界

1. `HR`：
   - 拥有 `page / info / add / update / setStatus`
2. 部门经理：
   - 仅拥有 `page / info`
   - 本人部门树范围内只读
3. 员工：
   - 无入口
   - 无任何主题17权限
