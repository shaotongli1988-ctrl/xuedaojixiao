## During Implementation

- Change summary：
  - 新增 `approval-flow` 后端资源：审批配置、配置节点、审批实例、实例节点、动作日志。
  - 新增 `approval-flow` controller 与 helper，落地冻结包定义的配置、详情、审批动作接口。
  - 将 `assessment.submit` / `promotion.submit` 接到自动审批实例创建链路。
  - 将 `assessment.approve` / `promotion.review` 接到 `assertManualReviewAllowed`，阻断已启用自动审批流对象绕过主链的旧入口。
  - 在 `approval-flow` 核心动作方法中补显式状态流断言，统一使用状态守卫保护 `approve / reject / transfer / withdraw / remind / resolve / fallback / terminate`。
  - 补充 service 测试与 controller 契约测试，覆盖正常、异常、边界、权限、状态、契约场景。
- Reused modules or patterns：
  - 复用 `performance` 模块既有 `entity/service/controller/test` 组织方式
  - 复用 `BaseSysMenuService`、`BaseSysPermsService`、`base_sys_user / department / role_menu / user_role` 作为权限和解析基础
- New dependencies and why existing tools were insufficient：
  - 无
- Verification run after each meaningful change：
  - 新增状态守卫断言后，执行定向后端测试。
  - 新增控制器契约测试后，再次执行定向后端测试。
  - 所有定向测试通过后执行 `npm run build`。
  - 额外执行统一交付守卫，用于识别剩余工具链级风险。
- Minimum regression matrix evidence：
  - 正常路径：`approval-flow.service.test.ts` 覆盖实例创建、通过、驳回、回退、终止。
  - 异常路径：覆盖非法审批人审批、非法状态提交旧入口等拒绝场景。
  - 边界路径：覆盖首节点未处理前撤回、待人工恢复、超时切 `manual_pending`。
  - 契约路径：`approval-flow.contract.test.ts` 覆盖 `configInfo/configSave/info/approve/reject/transfer/withdraw/remind/resolve/fallback/terminate` 控制器到 service 的参数转发。
- Permission checks：
  - 配置读取/保存仅校验 `performance:approvalFlow:configInfo` / `performance:approvalFlow:configSave`。
  - 审批动作分别校验 `approve/reject/transfer/withdraw/remind/resolve/fallback/terminate` 对应权限键。
  - `resolve/fallback/terminate` 额外要求 `HR` 身份。
- State-flow checks：
  - 审批实例状态固定为 `pending_resolution / in_review / manual_pending / approved / rejected / withdrawn / terminated`。
  - 动作守卫统一由 `allowedTransitionsByStatus` 与 `assertApprovalInstanceTransition` 提供。
  - `withdraw` 保留首节点未处理前可撤回的额外边界约束。
- Contract checks：
  - `approval-flow` 控制器接口已与冻结包定义的 11 个接口名对齐。
  - `assessment / promotion` 提交与手工审批兼容点已对齐后端服务调用约束。
