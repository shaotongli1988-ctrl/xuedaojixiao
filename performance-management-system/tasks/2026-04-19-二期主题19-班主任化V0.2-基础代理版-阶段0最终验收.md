## 交叉核验结果

### 1. 范围与非目标是否一致

- 结论：`是`
- 已一致部分：
  1. V0.2 仍归属于 `主题19`，不是拆出的新主题。
  2. V0.2 只新增代理骨架能力：
     - `teacherAgent`
     - `teacherAgentRelation`
     - `teacherAttribution`
     - `teacherAttributionConflict`
     - `teacherAgentAudit`
  3. V0.2 明确保持 V0.1 主链资源不变：
     - `teacherInfo`
     - `teacherFollow`
     - `teacherCooperation`
     - `teacherClass`
     - `teacherDashboard`
     - `teacherTodo`
  4. V0.2 明确不进入：
     - 结算体系
     - 分佣 / 返点
     - 账单 / 对账 / 打款
     - 复杂业绩报表
     - 合同签约链路
     - 绩效联动

### 2. 接口、权限、状态是否一致

- 结论：`是`
- 已一致部分：
  1. `teacherAgent` 固定为 `page / info / add / update / updateStatus / blacklist / unblacklist`。
  2. `teacherAgentRelation` 固定为 `page / add / update / delete`。
  3. `teacherAttribution` 固定为 `page / info / assign / change / remove`。
  4. `teacherAttributionConflict` 固定为 `page / info / create / resolve`。
  5. `teacherAgentAudit` 固定为 `page / info` 且只读。
  6. `teacherInfo` 只新增只读补充接口：
     - `attributionInfo`
     - `attributionHistory`
  7. 状态集合已固定为：
     - `teacherAgent.status`: `active / inactive`
     - `teacherAgent.blacklistStatus`: `normal / blacklisted`
     - `teacherAgentRelation.status`: `active / inactive`
     - `teacherAttribution.status`: `pending / active / removed / conflicted`
     - `teacherAttributionConflict.status`: `open / resolved / cancelled`
  8. 权限边界已固定为：
     - 管理层 / 超级管理员：全量
     - 部门负责人：部门树范围内维护代理、归因和冲突
     - 部门员工：本人范围查看和发起归因，不可处理冲突
     - 只读账号：只读

### 3. 关键规则与 V0.1 兼容约束是否一致

- 结论：`是`
- 已一致部分：
  1. 黑名单代理不可新增归因。
  2. 停用代理不可新增归因。
  3. 代理关系不得形成循环树。
  4. 一个 `teacherInfo` 同时只能有一条当前有效归因。
  5. 冲突未解决前，不允许新的归因结果直接生效。
  6. 归因冲突关闭后只允许保留一个最终有效归因。
  7. 代理体系不得改变 V0.1 的 `teacherCooperation` 状态机。
  8. 代理体系不得改变“仅 `partnered` 可建班”的前提。
  9. `teacherTodo` 继续只保留 `today / overdue` 两类派生桶。
  10. 只读账号查看班主任联系方式时继续沿用 V0.1 脱敏规则。
