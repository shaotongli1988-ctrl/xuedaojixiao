## 范围

1. 实现主题8前端页面：
   - 列表
   - 详情
   - 新增
   - 编辑
2. 前端只消费冻结允许的 `interview` 摘要字段，不渲染敏感候选人信息。
3. 落实页面权限显隐：
   - `performance:interview:page`
   - `performance:interview:info`
   - `performance:interview:add`
   - `performance:interview:update`
   - `performance:interview:delete`
4. 落实终态只读：
   - `completed`
   - `cancelled`

## 非目标

1. 不改后端接口、实体、服务和菜单
2. 不扩到招聘计划、简历池、职位标准、录用管理
3. 不展示手机号、邮箱、住址、简历全文、评语全文、录用决策全文
4. 不引入多面试官模型

## 验收口径

1. 页面只围绕 `interview page / info / add / update / delete` 工作。
2. 员工无入口时，前端页面应呈现无权限占位，不出现创建/编辑/删除动作。
3. 终态记录前端不可编辑，且保存请求不应把终态变更作为常规编辑入口暴露。
4. 列表、详情、表单只出现冻结允许字段。
5. 前端必须通过：
   - `corepack pnpm run type-check`
   - `corepack pnpm run build`
