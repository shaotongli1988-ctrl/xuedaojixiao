## 最小测试矩阵

| 维度 | 最小验证点 |
| --- | --- |
| 正常路径 | `HR` / 部门经理可新增录用、查看详情、将 `offered` 更新为 `accepted` 或 `rejected` |
| 异常路径 | 员工访问录用管理接口被拒绝；范围外经理访问或操作被拒绝 |
| 边界 | `accepted / rejected / closed` 终态不可再更新状态，不可重新打开 |
| 权限 | 部门经理仅可操作 `targetDepartmentId` 落在本人部门树范围内的录用单 |
| 状态 | 新增即 `offered`；仅允许 `offered -> accepted / rejected / closed` |
| 数据边界 | `hiring/info` 不返回简历全文、附件全文、面试评语全文；`hiring` 不自动反写主题8/12/15主状态 |
| 契约 | `add` 不要求 `interviewId`；`close` 必须保留关闭原因；`sourceType / sourceId` 仅作引用和快照来源 |

## 自动化与验证门槛

1. 后端最小 `jest` 必须覆盖：
   - `page / info / add / updateStatus / close`
   - 终态不可回退
   - 部门树权限拒绝
   - 员工拒绝
2. 前端最小门槛必须覆盖：
   - `type-check`
   - `build`
   - 列表 / 详情 / 新增页和状态动作显隐
3. smoke 最小覆盖：
   - `HR` 录用主链成功
   - 部门经理范围内成功、范围外拒绝
   - 员工无入口或接口拒绝
4. 主数据边界验证：
   - 录用详情不拉取简历全文
   - 录用完成不自动改写人才资产状态
   - 录用关闭不自动关闭面试或简历池
