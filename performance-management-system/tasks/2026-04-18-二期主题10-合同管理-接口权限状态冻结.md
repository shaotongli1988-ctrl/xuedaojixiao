## 已冻结资源边界

1. 资源名固定为 `contract`
2. 接口固定为：
   - `page`
   - `info`
   - `add`
   - `update`
   - `delete`
3. 状态固定为：
   - `draft`
   - `active`
   - `expired`
   - `terminated`
4. 权限固定为：
   - `HR` 全量管理
   - 部门经理无入口
   - 员工无入口

## 已冻结关键答案

1. 首批 API 只允许 `page / info / add / update / delete`，不新增 `sign / preview / archive / approve`。
2. 首批状态集合只保留台账状态，不引入 `signing / signed / approving` 等签署流状态。
3. 首批不开放员工侧入口，也不开放经理侧入口。
4. 详情只返回台账必要字段；附件全文、PDF 预览流、签名图片、签署轨迹、审批历史和甲乙方敏感证照 / 账户信息全部排除。
5. 首批不与主题5自动审批流耦合；若后续需要合同审批链，必须重新进入阶段0冻结。
