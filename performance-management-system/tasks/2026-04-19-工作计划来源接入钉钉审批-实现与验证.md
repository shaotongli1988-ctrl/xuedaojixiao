# 工作计划来源接入钉钉审批-实现与验证

## Before Starting
- Scope:
  为绩效域新增 `workPlan` 资源，支持手工创建工作计划、承接钉钉审批来源、拆分维护执行状态与来源审批状态，并提供页面入口与最小测试。
- Non-goals:
  不接入真实钉钉 SDK 鉴权、不编排真实审批流节点、不做消息通知、不把现有内部审批中心改造成钉钉流程引擎。
- Affected modules:
  `cool-admin-midway` 绩效模块后端、菜单与权限种子、迁移脚本；`cool-admin-vue` 工作计划页面、类型与服务；`performance-management-system/docs` API 与状态机文档。
- Acceptance criteria:
  1. 可手工创建、编辑、删除、开始、完成、取消工作计划。
  2. 可通过手工同步或回调承接钉钉审批来源，按实例 ID 幂等 upsert。
  3. `sourceStatus` 与执行 `status` 分离，未审批通过的钉钉来源计划不能开始。
  4. 菜单、权限、前端页面与后端接口对齐。
- Impact surface:
  实体、服务、控制器、菜单、权限种子、前端路由页面、类型、迁移脚本、本地 schema 兜底、测试与文档。
- Source of truth:
  用户确认的最小方案：“工作计划作为独立业务资源，支持钉钉审批来源同步，审批状态与执行状态分离”。
- Requirement conflicts or unclear points that must be resolved before coding:
  无；已明确不扩展为真实钉钉审批引擎。
- High-risk surfaces:
  新增接口契约、权限点、状态机、数据库表、回调兼容路径、本地 schema 初始化。
- Required delivery evidence:
  后端构建、前端 type-check/build、工作计划服务测试、文档更新。
- Required test matrix:
  正常路径：开始执行、来源同步 upsert。
  错误路径：未审批通过禁止开始。
  边界路径：计划开始日期晚于结束日期禁止保存。
- Key assumptions and constraints:
  钉钉仅作为来源系统；当前阶段回调用共享 token 兜底，不引入 OAuth / SDK。

## During Implementation
- Change summary:
  新增 `performance_work_plan` 实体、服务、控制器和页面；补 `/performance/work-plan` 菜单、权限和种子；新增正式迁移与本地 schema 兜底；补工作计划服务测试和契约文档。
- Reused modules or patterns:
  复用绩效模块既有 `BaseService + currentCtx/currentAdmin + 菜单权限 + 部门范围` 模式，以及前端绩效模块公共选项加载工具。
- New dependencies and why existing tools were insufficient:
  无新增依赖。
- Verification run after each meaningful change:
  修复后先跑 `npm run type-check`、`npm run build`；补测试后跑 `npx jest test/performance/work-plan.service.test.ts --runInBand`；最终再跑前后端构建。
- Normal-path evidence:
  工作计划可开始执行；钉钉回调可在无登录态下更新既有工作计划并返回规范化详情。
- Error-path evidence:
  `sourceType=dingtalkApproval` 且 `sourceStatus!=approved` 时，`start` 明确报错。
- Boundary-case evidence:
  `plannedStartDate > plannedEndDate` 时，更新与同步均被拒绝。
- Permission checks:
  页面/详情/新增/编辑/删除/开始/完成/取消/同步分别绑定 `performance:workPlan:*` 权限；回调接口改为 token 校验，不依赖登录态。
- State-flow checks:
  执行状态与来源状态拆分；已完成不回退；钉钉来源未通过前不可开始。
- Contract checks:
  前端服务、页面筛选、菜单权限与后端接口路径保持一致；新增回调头 `x-workplan-sync-token` 约定。
- Change-type verification mapping:
  API/契约变更对应前端 type-check 与构建；后端逻辑对应服务测试与后端构建；数据库变更对应迁移脚本与本地 schema 兜底。
- High-risk trigger changes:
  新表、回调兼容路径、状态机扩展、权限点新增。
- Compatibility or fallback impact:
  未接入真实钉钉时，仍可通过手工同步接口录入来源审批；回调 token 未配置时会拒绝外部同步。
- Privacy/PII review result:
  工作计划仅保存负责人/协作人 ID 与名称快照，不写入审批原文全文、通讯录敏感字段或鉴权密钥。
- Docs or operational updates made:
  更新 `04-API设计.md`、`05-状态机与流转规则.md`；新增迁移脚本和本地 schema 兜底；新增环境变量要求 `WORK_PLAN_DINGTALK_SYNC_TOKEN`。
