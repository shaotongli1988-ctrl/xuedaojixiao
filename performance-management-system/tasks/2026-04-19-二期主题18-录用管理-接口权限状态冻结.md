# 二期主题18-录用管理-接口权限状态冻结

## 基本信息

- 当前状态：done
- 负责人：Codex
- 创建时间：2026-04-19
- 最近更新时间：2026-04-19

## 结论

- 当前唯一正式结论：`录用管理接口、权限与状态已冻结`

## 已冻结资源边界

1. 资源名固定为 `hiring`
2. 首批最小接口固定为：
   - `page`
   - `info`
   - `add`
   - `updateStatus`
   - `close`
3. 首批不开放：
   - `delete`
   - `export`
   - `downloadAttachment`
   - `createEmployee`
   - `syncTalentAssetStatus`
4. 录用单首批允许手工创建，`add` 不要求必须传入 `interviewId`。

## 最小接口方向

1. 录用分页：`/admin/performance/hiring/page`
2. 录用详情：`/admin/performance/hiring/info`
3. 新增录用：`/admin/performance/hiring/add`
4. 更新录用状态：`/admin/performance/hiring/updateStatus`
5. 撤回 / 关闭录用：`/admin/performance/hiring/close`

## 权限冻结结论

1. `HR`
   - 全量 `page / info / add / updateStatus / close`
2. 部门经理
   - 仅本人部门树范围内 `page / info / add / updateStatus / close`
   - 数据范围固定按 `targetDepartmentId` 判定
3. 员工
   - 无入口
   - 无页面权限
   - 无动作权限
4. 首批不新增独立 `面试官 / 招聘专员 / Offer 审批人` 角色模型。

## 状态集合

1. `offered`
2. `accepted`
3. `rejected`
4. `closed`

## 合法流转

| 当前状态 | 动作 | 下一状态 | 允许角色 |
| --- | --- | --- | --- |
| 新建录用 | 保存 | `offered` | `HR` / 部门经理 |
| `offered` | 接受录用 | `accepted` | `HR` / 部门经理 |
| `offered` | 拒绝录用 | `rejected` | `HR` / 部门经理 |
| `offered` | 撤回 / 关闭 | `closed` | `HR` / 部门经理 |

## 状态补充规则

1. `accepted / rejected / closed` 为终态，首批不允许再修改回 `offered`。
2. 首批不开放“草稿”状态，新增录用即视为已发出录用决策。
3. 首批不开放“已入职”“已签约”“已撤销待重发”等扩展状态。
4. 若后续需要更细状态机，必须重新打开阶段0冻结。

## 数据边界补充规则

1. `hiring/info` 首批只返回录用决策所需字段和来源快照，不返回简历全文、附件全文或面试评语全文。
2. 如传入 `sourceType / sourceId`，只作为引用和快照来源，不改变来源主题的全文归属。
3. `close` 必须记录关闭原因，但首批不自动分发到其他主题。
