## 最小测试矩阵

| 维度 | 最小验证点 |
| --- | --- |
| 正常路径 | 资产管理员可完成采购入库、台账建档、领用归还、维护完成、调拨完成、盘点完成、折旧重算、报废执行，并在首页 / 报表看到同步结果 |
| 异常路径 | 普通员工访问资产接口被拒绝；部门负责人执行采购入库或折旧重算被拒绝；已报废资产再领用被拒绝 |
| 边界 | `received` 入库必须落台账；`available` 才能发起调拨；`scrapped` 不可恢复；折旧仅对未报废资产生效 |
| 权限 | 管理层全量；资产管理员全量；部门负责人部门树范围；财务只读只读；员工无入口 |
| 状态 | 采购入库、调拨、盘点、维护、报废各自动作链只允许走已冻结合法流转 |
| 数据边界 | 采购 / 供应商只允许弱引用；部门负责人不返回财务总账字段；财务只读无写权限 |

2. 前端最小门槛必须覆盖：
   - `type-check`
   - `build`

3. smoke 最小覆盖：
   - 管理层全量成功
   - 资产管理员全链成功
   - 部门负责人部门树范围成功
   - 财务只读只读成功且写动作拒绝
   - 普通员工拒绝
4. GUI 最小覆盖：
   - 采购入库 -> 台账 -> 领用 / 归还 -> 维护 -> 调拨 / 盘点 -> 折旧 -> 报废 -> 报表闭环
