## 最小测试矩阵

| 维度 | 最小验证点 |
| --- | --- |
| 正常路径 | 部门员工可新增班主任、录入跟进、标记合作、为已合作班主任建班，并在待跟进 / 看板看到本人范围数据 |
| 异常路径 | 跨部门访问或操作被拒绝；只读账号写操作被拒绝；未合作班主任建班被拒绝 |
| 边界 | 首次跟进后状态自动进入 `contacted`；`partnered` 前不可建班；`closed` 班级不可编辑、不可删除 |
| 权限 | 管理层全量；部门负责人部门树范围；部门员工本人范围；只读账号只读且联系方式脱敏 |
| 状态 | `uncontacted -> contacted -> negotiating -> partnered -> terminated` 合法；班级 `draft -> active -> closed` 合法 |
| 数据边界 | 看板只返聚合统计；详情不返回沟通截图原文、合作资料二进制、代理 / 绩效 / 结算字段；只读账号联系方式脱敏 |

## 自动化与验证门槛

1. 后端最小 `jest` 必须覆盖：
   - `teacherInfo / teacherFollow / teacherCooperation / teacherClass / teacherDashboard / teacherTodo`
   - 本人 / 本部门 / 全量 / 只读四类范围
   - 已合作前不可建班
   - `closed` 班级终态锁定
2. 前端最小门槛必须覆盖：
   - `type-check`
   - `build`
   - 看板、列表、详情、待办和班级页面主链
3. smoke 最小覆盖：
   - 管理层全量成功
   - 部门员工本人范围成功
   - 只读账号只读成功且写操作拒绝
   - 跨部门拒绝
4. GUI 最小覆盖：
   - 资源录入 -> 新增跟进 -> 标记合作 -> 建班 -> 看板 / 待办链路
