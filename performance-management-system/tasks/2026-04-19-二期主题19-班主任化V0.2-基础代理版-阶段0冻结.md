## 与 V0.1 的关系

1. `主题19 V0.1` 已完成并落地以下资源主链：
   - `teacherInfo`
   - `teacherFollow`
   - `teacherCooperation`
   - `teacherClass`
   - `teacherDashboard`
   - `teacherTodo`
2. `主题19 V0.2` 不是替换 V0.1，而是在不破坏既有主链的前提下，新增“代理渠道归属层”。
3. V0.2 的新增能力只能围绕：
   - 代理主体管理
   - 代理关系管理
   - 资源归因
   - 归因冲突处理
   - 审计留痕
4. V0.2 不得反向改写 V0.1 已冻结的合作状态机、建班前置条件、待办口径和只读脱敏规则。

## 业务目标

1. 在班主任资源主链上新增代理渠道骨架，使 `teacherInfo` 可以归属于代理主体或直营。
2. 支持最小代理树关系管理，为后续渠道归因、业绩统计和结算扩展打基础。
3. 支持一个班主任资源只有一个当前有效归因，重复认领必须进入冲突处理。
4. 支持代理停用、黑名单和关键操作审计留痕，避免渠道归因链路失控。

## 非目标

1. 不做分佣规则
2. 不做返点规则
3. 不做账单
4. 不做对账
5. 不做打款记录
6. 不做复杂排行和复杂经营报表
7. 不做绩效管理联动
8. 不做合同签约主链
9. 不做 Excel 导入导出
10. 不做短信提醒、外部自动化通知

## 资源关系模型

1. `teacherInfo` 继续代表班主任资源本体，不被代理体系替代。
2. `teacherAgent` 代表代理主体，是 `teacherInfo` 的归属目标之一。
3. `teacherAgentRelation` 维护代理主体之间的上下级结构，不直接挂班主任资源。
4. `teacherAttribution` 作为桥接资源，表达“某个班主任当前归属于哪个代理主体或直营”。
5. `teacherAttributionConflict` 负责承接一个班主任被多个代理重复认领时的冲突态。
6. `teacherAgentAudit` 对代理主体、代理关系、归因、冲突处理等关键动作自动留痕。

   - 仅查看授权范围内代理、归因、冲突结果和审计日志
   - 无新增、更新、处理权限

## 关键业务规则

### 代理主体规则

1. 黑名单代理不可新增归因。
2. 停用代理不可新增归因。
3. 代理主体默认不开放物理删除，V0.2 只做停用。
4. 代理主体历史关联数据必须保留。

### 代理关系规则

1. 不允许形成循环父子关系。
2. 停用代理不能作为新的父级或子级关系目标。
3. 代理关系新增、调整、失效都必须写审计日志。

### 资源归因规则

1. 一个 `teacherInfo` 同时只能存在一条当前生效归因。
2. 未归因班主任资源可归直营或归代理主体。
3. 已归因资源再次归因时，不得直接覆盖，必须先判断是否冲突。
4. 黑名单或停用代理不能接收新归因。
5. 已 `terminated` 的班主任资源不允许新建代理归因。
6. 已存在班级的班主任资源允许调整归因，但必须完整留痕。

### 冲突处理规则

1. 同一个 `teacherInfo` 被多个代理主张时进入冲突态。
2. 冲突未解决前，不允许新归因生效。
3. 冲突处理结束后，只允许保留一个最终有效归因。
4. 冲突关闭时必须记录处理人、处理时间和处理说明。

### 审计规则

以下动作必须自动留痕：

1. 新增 / 编辑代理主体
2. 代理主体启用 / 停用
3. 代理主体黑名单 / 移出黑名单
4. 代理关系新增 / 调整 / 失效
5. 资源归因建立 / 调整 / 移除
6. 冲突创建 / 处理

补充约束：

1. `teacherAgentAudit` 只读
2. 前端不得直接新增或修改审计记录

### 与 V0.1 主链兼容规则

1. `teacherTodo` 仍然只允许 `today / overdue` 两类派生桶。
2. 代理体系不得改变 `teacherCooperation` 原有状态集合。
3. 代理体系不得改变“仅 `partnered` 可建班”的原有前提。
4. 代理体系不得改变 `closed class` 不可编辑、删除、重开的原有规则。

## 数据边界

1. V0.2 返回的是代理主体、代理关系、归因、冲突和审计字段，不返回结算金额字段。
2. 不返回分佣、返点、账单、对账或打款记录。
3. 看板如果补充代理统计，首批只允许聚合统计，不允许复杂报表。
4. 只读账号在查看班主任资源详情时，仍继续沿用 V0.1 联系方式脱敏规则。

## 最小验收口径

### 正常路径

1. 可新增代理主体。
2. 可维护代理上下级关系。
3. 可给未归因班主任资源建立归因。
4. 可查看班主任当前归因和归因历史。
5. 可处理归因冲突并生成唯一有效归因。
6. 可在班主任详情页查看代理信息与审计记录。

### 异常路径

1. 停用代理归因被拒绝。
2. 黑名单代理归因被拒绝。
3. 循环代理关系被拒绝。
4. 冲突未解决前重复归因被拒绝。
5. 非法冲突处理被拒绝。

### 权限路径

1. 管理层全量成功。
2. 部门负责人仅部门树范围内成功。
3. 部门员工无冲突处理权限。
4. 只读账号只能查看。

### 边界路径

1. 已 `terminated` 资源新增归因被拒绝。
2. 已建班资源调整归因成功，但审计记录完整。
3. 代理停用后，历史归因仍可查看。
4. 删除动作默认不可见或不可调用。
