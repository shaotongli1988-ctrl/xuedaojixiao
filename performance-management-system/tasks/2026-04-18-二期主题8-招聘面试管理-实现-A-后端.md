## 需求摘要

- 本次要解决什么问题：把主题8冻结后的 `interview` 后端主链真正落到代码，提供 `page / info / add / update / delete` 最小可联调接口。
- 目标效果是什么：`HR` 可全量管理，部门经理只能操作 `departmentId` 在本人部门树范围内的面试记录，员工无权访问；`completed / cancelled` 终态不可编辑，删除只允许 `scheduled` 且只保留给 `HR`。
- 影响范围大概有哪些：绩效模块 `interview` entity / service / controller / jest 测试，以及本任务卡回写。

## 当前结论

- 当前已经确认的结论：
  - 资源名固定为 `interview`，接口路径固定为 `/admin/performance/interview/page|info|add|update|delete`。
  - 候选人信息仍为手工文本录入，不接简历池，不新增候选人主数据绑定。
  - 面试状态固定为 `scheduled / completed / cancelled`；`completed / cancelled` 为终态，不允许补录或修改业务字段。
  - 部门经理范围固定按 `departmentId` 判定，不按面试官所属部门推导。
  - `interviewerId` 首批固定为单人字段，不扩展多人面试官模型。
  - 删除权限只保留 `HR`，实现上通过 `performance:interview:delete` 权限键收口。
- 当前阶段为什么可以关闭：后端主链已实现，主题8最小后端自动化验证与后端 `build` 均通过，且本轮没有发现未关闭 `P0 / P1`。
