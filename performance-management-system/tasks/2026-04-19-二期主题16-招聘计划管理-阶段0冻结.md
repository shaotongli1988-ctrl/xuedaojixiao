## 当前已冻结边界

1. 资源名固定为：`recruitPlan`
2. 页面入口固定为：
   - 路由：`/performance/recruit-plan`
   - `viewPath`：`modules/performance/views/recruit-plan/index.vue`
3. 首批动作固定为：
   - `page / info / add / update / delete / import / export / submit / close / void / reopen`
4. 状态固定为：
   - `draft / active / voided / closed`
5. 权限固定为：
   - `performance:recruitPlan:page`
   - `performance:recruitPlan:info`
   - `performance:recruitPlan:add`
   - `performance:recruitPlan:update`
   - `performance:recruitPlan:delete`
   - `performance:recruitPlan:import`
   - `performance:recruitPlan:export`
   - `performance:recruitPlan:submit`
   - `performance:recruitPlan:close`
   - `performance:recruitPlan:void`
   - `performance:recruitPlan:reopen`
6. 字段边界固定为招聘计划摘要字段，不返回候选人名单、预算明细、录用结果、简历全文或面试评语全文。
7. 角色边界固定为：
   - `HR`：全量可见、可维护
   - 部门经理：本人部门树范围内可见、可维护
   - 员工：无入口、无读写权限
8. `submit` 表示备案生效，不表示进入审批流。
9. `delete` 仅允许 `draft` 且无下游引用。
10. 导出只允许摘要字段，导入只允许招聘计划摘要模板字段。
11. 首批允许 `recruitPlan` 通过 `jobStandardId` 弱引用职位标准，但不因此把主题17变成招聘流程前置中心。
12. 关闭、作废、重新开启都不自动改写主题8、15、18主状态，也不要求强绑定职位标准。
