# 二期主题18-录用管理-主数据边界冻结

## 结论

- 当前唯一正式结论：`录用管理主数据边界已冻结`

## 全文 / 摘要 / 快照职责冻结

1. `resumePool`
   - 全文主数据拥有者
   - 负责：
     - 简历全文
     - 联系方式
     - 附件摘要与下载入口
     - 候选人来源和原始投递材料
   - 不负责：
     - 面试评语全文
     - 录用决策全文
2. `interview`
   - 面试业务记录拥有者
   - 负责：
     - 面试安排
     - 面试状态
     - 面试评语全文
     - 面试结果摘要
   - 不负责：
     - 简历全文
     - 录用决策全文
     - 员工入职主数据
3. `talentAsset`
   - 招聘侧未入职优秀人才摘要主链拥有者
   - 负责：
     - 标签
     - 来源摘要
     - 跟进摘要
     - 招聘侧优秀人才池视图
   - 不负责：
     - 简历全文
     - 联系方式全文
     - 面试评语全文
     - 录用决策全文
4. `hiring`
   - 录用决策主链拥有者
   - 负责：
     - 录用状态
     - Offer 发出 / 接受 / 拒绝 / 关闭时间线
     - 录用决策全文
     - 关闭原因
     - 候选人业务快照和来源引用
   - 不负责：
     - 简历全文
     - 附件全文
     - 面试评语全文
     - 员工主数据

## 快照与摘要冻结结论

1. `hiring` 中允许保存：
   - `candidateName`
   - `targetDepartmentId`
   - `targetPosition`
   - `sourceType`
   - `sourceId`
   - 来源状态快照
2. `hiring` 中不允许保存或接管：
   - 简历正文
   - 联系方式全文主拷贝
   - 附件文件内容
   - 面试评语全文
3. `resumePool / talentAsset / interview` 若展示录用信息，首批只允许保留：
   - `linkedHiringId`
   - `hiringStatusSummary`
   - `lastHiringUpdateTime`
4. 其他主题首批不允许保存 `hiring` 决策全文副本。

## 允许的单向流转

1. 当前首批唯一保证的创建方式：`manual -> hiring`
2. 若后续单独评审通过，可允许：
   - `resumePool -> hiring`
   - `talentAsset -> hiring`
   - `interview -> hiring`
3. 上述跨主题流转即使后续开放，也只能是：
   - 一次性复制业务快照
   - 记录来源引用 `sourceType / sourceId`
   - 不改写来源主题的全文归属

## 禁止的自动反写

1. 禁止 `hiring accepted` 自动把 `resumePool` 置为归档或关闭。
2. 禁止 `hiring accepted / rejected / closed` 自动把 `talentAsset` 主状态改写为录用结果状态。
3. 禁止 `hiring` 自动修改 `interview` 状态、评语全文或候选人文本主字段。
4. 禁止 `hiring` 自动创建员工、人事档案、合同或入职记录。
5. 禁止把 `talentAsset` 反向升级为全文主数据中心。

## 录用完成后的人才资产反写结论

1. 首批不自动反向更新 `talentAsset` 主状态。
2. 如业务后续需要“录用完成同步人才资产状态”，必须重新打开阶段0冻结，单独评审：
   - 状态映射
   - 回滚语义
   - 审计与人工纠偏方式

## 必须后续单独评审的跨主题动作

1. 主题15新增“一键发起录用”
2. 主题12新增“发起录用”入口
3. 主题8新增“面试通过自动建录用单”
4. `hiring accepted` 自动关闭简历池或自动沉淀人才资产状态
5. `hiring accepted` 自动创建员工 / 入职 / 合同 / 薪资记录
6. 在 `hiring` 详情中直接拉取简历附件下载或面试评语全文
