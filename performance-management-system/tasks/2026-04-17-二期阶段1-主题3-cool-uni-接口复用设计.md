## 页面级复用说明

### 登录

1. 共享基础层已提供验证码、登录、`permmenu` 三段式链路。
2. 当前冻结口径要求移动端直接复用，不允许引入短信登录、手机号快捷登录或移动端专用 token 协议。
3. 本页不存在绩效业务接口缺口。

### 工作台

1. `permmenu` 已可决定首批入口显隐。
2. `assessment/page`、`goal/page`、`feedback/page` 都返回 `pagination.total`，可通过 `page=1,size=1` 的轻量请求生成摘要计数。
3. 当前工作台定义为“入口 + 待办摘要”，不是驾驶舱；因此不能把缺少单接口聚合视为缺口。
4. 只要不把工作台扩成跨模块统计看板，当前接口即可覆盖。

### 我的考核

1. `assessment/page` 已通过 `mode=my` 固化“当前登录人作为 employeeId”的查询口径。
2. `assessment/update` 已支持员工在 `draft / rejected` 态保存自评草稿。
3. `assessment/submit` 已固定为员工在 `draft` 态触发。
4. 当前页面不存在接口缺口。

### 待我审批

1. `assessment/page` 已通过 `mode=pending` 固化“可审批且 `status=submitted`”的查询口径。
2. `assessment/approve` 与 `assessment/reject` 已对应状态机审批动作。
3. `HR` 虽可见该菜单，但冻结范围已排除 `HR` 移动端，因此不构成移动端接口差异。
4. 当前页面不存在接口缺口。

### 目标

1. `goal/page`、`goal/info` 已覆盖列表与详情。
2. `goal/update` 已覆盖现有事实源允许的本人目标编辑。
3. `goal/progressUpdate` 已覆盖移动端高频动作，且服务端自动承接 `draft -> in-progress -> completed`。
4. 当前页面不存在接口缺口。

### 360 环评

1. `feedback/page` 作用域已同时覆盖“本人被评价任务”和“本人作为评价人待提交任务”。
2. `feedback/info` 会为员工返回 `currentUserRecord`，并按角色裁剪 `feedbackUsers` 明细。
3. `feedback/summary` 会为员工返回汇总，不返回他人单条 `records[]` 内容，符合脱敏规则。
4. `feedback/submit` 已覆盖单人单次提交、截止时间和关闭态校验。
5. 当前页面不存在接口缺口。

## 100% 可复用清单

1. 登录：共享基础层现有 `captcha / login / permmenu` 链路可直接复用。
2. 工作台：通过 `permmenu + assessment / goal / feedback page` 的轻量请求组合可实现，无需聚合新接口。
3. 我的考核：现有 `assessment` 四个接口已覆盖列表、详情、草稿保存、提交。
4. 待我审批：现有 `assessment` 四个接口已覆盖列表、详情、通过、驳回。
5. 目标：现有 `goal` 四个接口已覆盖列表、详情、编辑、更新进度。
6. 360 环评：现有 `feedback` 四个接口已覆盖任务查看、反馈提交、汇总查看。
