## 实现边界

1. 本轮只允许实现：
   - `teacherAgent page / info / add / update / updateStatus / blacklist / unblacklist`
   - `teacherAgentRelation page / add / update / delete`
2. 本轮必须遵守：
   - `teacherAgent.status` 固定为 `active / inactive`
   - `teacherAgent.blacklistStatus` 固定为 `normal / blacklisted`
   - `teacherAgentRelation.status` 固定为 `active / inactive`
   - 管理层 / 超级管理员全量
   - 部门负责人仅部门树范围内维护
   - 部门员工与只读账号无写权限
3. 本轮必须兜底：
   - 黑名单代理不可作为新归因目标
   - 停用代理不可作为新归因目标
   - 代理关系不得形成循环树
   - 停用代理不能作为新的父级或子级关系目标
4. 本轮允许为后续窗口预留：
   - 代理主体、代理关系变更所需的审计留痕写入点
   - 但 `teacherAgentAudit page / info` 对外接口不在本卡实现范围内
5. 本轮不允许：
   - 新增代理主体物理删除
   - 扩到 `teacherAttribution / teacherAttributionConflict / teacherAgentAudit` 对外查询接口
   - 扩到结算、分佣、返点、账单、对账、打款、复杂报表、绩效、合同链路
   - 反向改写 V0.1 的合作、建班、待办和脱敏规则
