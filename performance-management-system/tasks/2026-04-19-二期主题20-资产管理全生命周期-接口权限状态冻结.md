## 已冻结资源边界

1. 资源名固定为：
   - `assetInfo`
   - `assetAssignment`
   - `assetMaintenance`
   - `assetProcurement`
   - `assetTransfer`
   - `assetInventory`
   - `assetDepreciation`
   - `assetDisposal`
   - `assetDashboard`
   - `assetReport`
2. 首批接口固定为：
   - `assetInfo page / info / add / update / delete / updateStatus`
   - `assetAssignment page / add / update / return / markLost / delete`
   - `assetMaintenance page / add / update / complete / cancel / delete`
   - `assetProcurement page / info / add / update / submit / receive / cancel`
   - `assetTransfer page / info / add / update / submit / complete / cancel`
   - `assetInventory page / info / add / update / start / complete / close`
   - `assetDepreciation page / summary / recalculate`
   - `assetDisposal page / info / add / update / submit / approve / execute / cancel`
   - `assetDashboard summary`
   - `assetReport summary / page / export`
3. 首批不开放：
   - `supplierMaster`
   - `purchaseOrderApprove`
   - `voucherPosting`
   - `rfidSync`
   - `mobileScan`

## 权限冻结结论

1. 管理层 / 超级管理员：全量。
2. 资产管理员：资产全生命周期全量管理。
3. 部门负责人：仅本人部门树范围内可查看台账、领用归还、维护、调拨、盘点和报废单，不拥有采购入库和折旧重算权限。
4. 财务只读：可查看首页、报表、折旧和采购入库金额摘要，但无写权限。
5. 普通员工：首批无资产管理后台入口。

## 合法流转

### 资产主状态

| 当前状态 | 动作 | 下一状态 | 允许角色 |
| --- | --- | --- | --- |
| 新增采购入库并确认入库 | 创建资产 | `available` | 管理层 / 超级管理员 / 资产管理员 |
| `available` | 新增领用 | `assigned` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `assigned` | 归还 | `available` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `available` / `assigned` | 开始维护 | `maintenance` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `maintenance` | 完成维护 | `available` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `available` | 提交调拨并执行完成 | `inTransfer -> available` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `available` | 开始盘点 | `inventorying` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| `inventorying` | 完成 / 关闭盘点 | `available` | 管理层 / 超级管理员 / 资产管理员 / 部门负责人 |
| 任意非终态 | 标记丢失 | `lost` | 管理层 / 超级管理员 / 资产管理员 |
| 任意非终态 | 执行报废 | `scrapped` | 管理层 / 超级管理员 / 资产管理员 |

### 子单据状态

1. 采购入库：`draft -> submitted -> received`，或 `draft / submitted -> cancelled`
2. 调拨：`draft -> submitted -> inTransit -> completed`，或 `draft / submitted / inTransit -> cancelled`
3. 盘点：`draft -> counting -> completed -> closed`
4. 报废：`draft -> submitted -> approved -> scrapped`，或 `draft / submitted / approved -> cancelled`

## 状态补充规则

1. `scrapped` 资产不可再次领用、维护、调拨、盘点或恢复。
2. `lost` 资产不可发起调拨和报废之外的新操作。
3. `received` 采购入库必须落资产台账主记录，不允许只停留在单据层。
4. 调拨只允许基于 `available` 资产发起。
5. 折旧不作为资产主状态，仅作为月度快照和汇总视图。

## 数据边界补充规则

1. 主题20允许弱引用 `purchaseOrderId / supplierId` 或快照字段，但不接管主题11主数据维护。
2. 部门负责人不返回折旧重算参数、财务凭证号、税务字段和总账字段。
3. 财务只读可见金额、折旧摘要和报表导出，不可见写动作。
4. 普通员工无后台页面入口，不返回主题20主链数据。
5. 资产领用审批分层正式事实源以 [2026-04-19-资产领用审批分层规则正式冻结.md](/Users/shaotongli/Documents/xuedao/performance-management-system/tasks/2026-04-19-资产领用审批分层规则正式冻结.md) 为准；若未来进入系统审批化，必须先同步本卡、主题20主卡和最终验收卡。
