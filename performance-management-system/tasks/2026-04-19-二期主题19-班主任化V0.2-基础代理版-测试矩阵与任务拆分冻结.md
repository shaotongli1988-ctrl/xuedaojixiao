## 最小测试矩阵

| 维度 | 最小验证点 |
| --- | --- |
| 正常路径 | 可新增代理主体、维护代理关系、建立班主任归因、查看归因历史、处理冲突并形成唯一有效归因 |
| 异常路径 | 停用代理归因被拒绝；黑名单代理归因被拒绝；循环代理关系被拒绝；冲突未解决前重复归因被拒绝 |
| 边界 | 一个 `teacherInfo` 同时只能有一条当前生效归因；已 `terminated` 资源不可新增归因；已建班资源可调整归因但必须留痕 |
| 权限 | 管理层全量；部门负责人部门树范围；部门员工不可处理冲突；只读账号只读 |
| 状态 | `teacherAgent active/inactive`、`blacklist normal/blacklisted`、`teacherAttribution pending/active/removed/conflicted`、`teacherAttributionConflict open/resolved/cancelled` 合法 |
| 数据边界 | 不返回分佣、返点、账单、对账、打款和复杂报表字段；审计记录只读；`teacherTodo` 仍只保留 `today / overdue` |

## 自动化与验证门槛

1. 后端最小 `jest` 必须覆盖：
   - `teacherAgent / teacherAgentRelation / teacherAttribution / teacherAttributionConflict / teacherAgentAudit`
   - 归因唯一性
   - 停用代理和黑名单代理不可归因
   - 代理关系防循环
   - 冲突创建与处理
   - 管理层 / 部门负责人 / 部门员工 / 只读账号四类权限
2. 前端最小门槛必须覆盖：
   - `type-check`
   - `build`
   - 代理主体列表 / 详情
   - 代理关系维护页
   - 班主任详情归因区块
   - 冲突列表 / 详情 / 处理页
   - 审计记录查看页或查看抽屉
3. smoke 最小覆盖：
   - 管理层新增代理主体成功
   - 管理层建立归因成功
   - 停用代理归因拒绝
   - 黑名单代理归因拒绝
   - 部门负责人范围内归因成功
   - 部门员工冲突处理拒绝
   - 只读账号写操作拒绝
4. GUI 最小覆盖：
   - 代理主体新增 / 编辑
   - 代理关系维护
   - 班主任详情查看归因
   - 触发冲突并处理
   - 查看审计记录
