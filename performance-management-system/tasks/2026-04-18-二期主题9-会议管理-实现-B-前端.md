  - 新增类型、service、组件、页面后执行前端 `type-check`。

  - `type-check` 通过后执行前端 `build`。
- Minimum regression matrix evidence:
  - 正常：具备列表查询、新增、编辑、详情、会议级签到入口。
  - 权限：页面入口、详情、新增、编辑、签到均绑定 `performance:meeting:*` 权限键显隐。
  - 状态：签到按钮仅在 `in_progress` 显示；`completed / cancelled` 不显示编辑按钮。
  - 脱敏：详情与列表均不展示参与人名单，只展示 `participantCount` 摘要。
- Permission checks:

  - 编辑权限：`performance:meeting:update`
  - 签到权限：`performance:meeting:checkIn`
- State-flow checks:
  - 创建表单固定默认 `scheduled`
  - `scheduled` 编辑时只允许切到 `scheduled / in_progress / cancelled`
  - `in_progress` 编辑时只允许切到 `in_progress / completed`
  - `completed / cancelled` 终态不提供编辑按钮
- Contract checks:
  - 请求字段按 `04-API设计.md` 对齐 `title / code / type / description / startDate / endDate / location / organizerId / participantIds / status`
  - 详情页只消费冻结的摘要字段，重点依赖 `participantCount`
- High-risk trigger changes:
