## API / 权限 / 状态 / 数据边界冻结摘要

### API 方向

1. 主题14 统一冻结三组资源：
   - `courseRecite`
   - `coursePractice`
   - `courseExam`
2. 首批标准接口固定为：
   - `/admin/performance/courseRecite/page`
   - `/admin/performance/courseRecite/info`
   - `/admin/performance/courseRecite/submit`
   - `/admin/performance/coursePractice/page`
   - `/admin/performance/coursePractice/info`
   - `/admin/performance/coursePractice/submit`
   - `/admin/performance/courseExam/summary`
3. 首批只冻结文本提交，不冻结音频、图片、附件、流式输出和多轮对话接口。

### 权限边界

1. 主题14 固定为员工本人自助学习能力。
2. 员工拥有：
   - 背诵任务列表 / 详情 / 提交
   - 练习任务列表 / 详情 / 提交
   - 考试 / 结果摘要
3. `HR` 与部门经理首批无主题14接口权限；如需课程侧摘要，继续走主题7。

### 状态边界

1. 背诵 / 练习任务状态固定为：
   - `pending`
   - `submitted`
   - `evaluated`
2. 结果摘要状态固定为：
   - `locked`
   - `pending`
   - `passed`
   - `failed`
3. 首批不冻结补考、重做、多轮追问或人工改分状态。

### 数据边界

1. 只允许返回当前员工本人任务与结果摘要。
2. 允许返回任务摘要、任务内容摘要、本人提交文本和结果摘要。
3. 禁止返回：
   - 外部 `AI` 供应商 / 模型 / Prompt 轨迹
   - 试卷全文、标准答案、评分规则全文
   - 其他员工的任务与结果
   - 音频、图片、附件原文

## 真实 AI 能力调用边界

1. 当前事实源不足以冻结真实 `AI` 能力调用。
2. 因此首批只允许冻结：
   - 服务端提交接口
   - 结果摘要字段结构
   - 厂商隐藏原则
3. 当前不允许冻结：
   - 供应商名称
   - 模型名称
   - Prompt 编排协议
   - 实时 / 流式对话协议
   - 客户端直连外部 `AI`

## 测试矩阵与后续实现拆分

### 最小测试矩阵

1. 正常路径：
   - 员工可从课程关联入口进入本人背诵 / 练习任务
   - 员工可查看详情并提交文本内容
   - 员工可查看考试 / 结果摘要
2. 异常路径：
   - `HR` / 部门经理访问主题14接口被拒绝
   - 员工访问他人任务或越权 `courseId` 被拒绝
3. 边界：
   - 只允许文本提交
   - `evaluated` 任务不可再次提交
   - 当前结果摘要在前置条件未满足时返回 `locked`
4. 权限：
   - 员工有主题14自助权限，但仍无主题7课程管理后台权限
5. 状态：
   - `pending -> submitted -> evaluated` 合法
   - 结果摘要只读返回 `locked / pending / passed / failed`

### 后续实现拆分

1. 串行步骤 1：固定 `courseRecite / coursePractice / courseExam` 的字段、状态、权限与脱敏边界。
2. 串行步骤 2：实现课程关联任务与结果摘要的后端只读 / 提交接口骨架。
3. 并行步骤 3：
   - 后端背诵 / 练习读取与提交
   - 后端考试 / 结果摘要只读接口
   - 前端课程关联入口与背诵 / 练习 / 结果摘要页面
4. 串行步骤 4：补齐本人范围回归、状态回归、主题7边界回归和文档收口。
