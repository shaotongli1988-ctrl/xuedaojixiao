# 权限矩阵

## 文档职责

这份文档是“权限规则”的唯一事实源。

它只负责定义：

1. 角色
2. 菜单权限
3. 页面按钮权限
4. 接口权限
5. 数据范围权限

## 不负责内容

这份文档不负责定义：

- 页面字段布局
- 接口字段格式
- 脱敏算法细节

脱敏细节以 `12-数据权限与脱敏规则.md` 为准。

## 权限命名规则

沿用当前 Cool 项目既有风格，统一使用：

`performance:<resource>:<action>`

示例：

- `performance:assessment:page`
- `performance:assessment:approve`
- `performance:goal:progressUpdate`
- `performance:salary:info`

## 角色定义

| 角色 | 说明 |
| --- | --- |
| HR 管理员 | 全局绩效配置、归档和敏感数据管理角色 |
| 部门经理 | 团队绩效审批和跟进角色 |
| 员工 | 个人绩效查看和自评角色 |

## 接口权限

### HR 管理员

- 拥有绩效模块全部查看类接口权限。
- 拥有薪资和指标配置接口权限。
- 拥有评估单新增、编辑、删除和归档管理权限。
- 首期不拥有评估审批动作权限。
- 首期独占薪资管理入口和金额维护权限。

### 部门经理

- 可访问：
  - 团队评估分页、详情、审批
  - 团队目标分页、更新进度
  - PIP 启动、跟进、完成、关闭
  - 晋升提交、评审
  - 环评相关接口
- 不可访问：
  - 指标库配置
  - 全量薪资分页
  - 薪资详情和薪资调整接口

### 员工

- 可访问：
  - 我的考核列表和详情
  - 本人自评提交
  - 本人目标查看和进度更新
  - 指派给自己的环评反馈提交
- 不可访问：
  - 他人评估单
  - 审批接口
  - 薪资管理后台
- 不可创建评估单
- 不可访问任何薪资管理接口

## 落地规则

1. 前端用 `v-permission` 和 `_permission` 控制按钮。
2. 后端必须再次校验角色和数据范围。
3. 菜单可见不等于数据可见，列表查询必须附带服务端数据范围条件。
4. 部门经理的数据范围统一按组织架构部门树计算，不单独支持跨部门兼任授权。
