## 测试目标

为首期绩效域模块提供最小可交付回归矩阵，保证下面 6 类高风险问题不会在首期上线前漏测：

1. 接口字段和页面字段不一致
2. 状态机流转错误
3. 权限显隐与后端鉴权不一致
4. 敏感数据越权返回
5. 驾驶舱统计口径错误
6. 关键动作执行后页面状态未刷新

## 最小回归矩阵

### 1. 绩效驾驶舱

#### 正常路径

- 能按默认周期加载驾驶舱数据
- KPI 卡片有值且字段完整
- 部门筛选后数据刷新
- 图表无空指针报错

#### 异常路径

- 无权限用户访问驾驶舱接口时被拒绝
- 无数据周期返回空态，不报错

#### 边界

- 部门无任何评估数据时，分布图返回空数组
- 周期内只有一条数据时，均分和分布仍可计算
- 首期接口返回中不包含跨模块保留指标字段，前端不依赖 `reservedMetrics`

### 2. 评估单主流程

#### 正常路径

- HR 或经理可以创建评估单
- 员工可查看本人评估单详情
- 草稿评估单可提交
- 已提交评估单可被部门经理审批通过
- 已提交评估单可被部门经理审批驳回

#### 异常路径

- 员工不能创建评估单
- 非草稿不能再次提交
- 非已提交状态不能执行审批
- HR 不能执行审批通过或驳回
- 不存在的评估单返回明确错误

#### 边界

- 周期重复但编号不同的评估单可区分
- 评分项为空时禁止提交或按设计规则拒绝
- 前端手工传入 `grade` 时，服务端不接受直接覆盖最终等级

### 3. 我的考核 / 已发起考核 / 待我审批三视图一致性

#### 正常路径

- 我的考核只展示当前登录人数据
- 已发起考核只展示自己发起或可管理范围数据
- 待我审批只展示 `submitted` 且当前人有审批权的数据
- 我的考核默认按 `updateTime desc` 排序
- 已发起考核默认按 `createTime desc` 排序
- 待我审批默认按 `submitTime asc` 排序

#### 异常路径

- 员工不能看到待我审批页面数据
- 经理不能越权看到非所辖部门考核
- HR 在待我审批页面不能执行审批按钮

#### 边界

- 同一评估单状态变化后，三个列表展示同步变化
- 同一条评估单可同时出现在“已发起考核”和“待我审批”中，但展示原因必须符合视角规则
- 经理可看到直接管理部门及其下级部门数据
- 经理不能看到跨部门兼任场景下未授权部门数据

### 4. 目标地图

#### 正常路径

- 可创建目标
- 可编辑目标
- 可更新目标进度
- 目标完成后状态变更正确

#### 异常路径

- 已完成目标不能继续更新进度
- 非本人或无权限用户不能修改目标

#### 边界

- 当前值等于目标值时，允许进入完成状态
- 当前值超过目标值时，按设计规则处理且不报错

### 5. 指标库

#### 正常路径

- HR 可新增、编辑、删除指标
- 指标列表按状态筛选有效

#### 异常路径

- 非 HR 访问新增或编辑接口被拒绝
- 重复指标编码被拒绝

### 5.1 导出能力

#### 正常路径

- `HR` 可执行绩效评估导出
- `HR` 和 `部门经理` 可执行目标地图导出

#### 异常路径

- `部门经理` 不可执行绩效评估导出
- `员工` 不可执行任何导出
- 首期访问 `薪资`、`环评`、`PIP`、`晋升` 导出能力时，应无入口或被拒绝

#### 敏感数据测试

- 绩效评估导出仅包含：
  - `code`
  - `employeeName`
  - `departmentName`
  - `periodType`
  - `periodValue`
  - `assessorName`
  - `status`
  - `targetCompletion`
  - `totalScore`
  - `grade`
  - `submitTime`
  - `approveTime`
- 绩效评估导出不包含：
  - `selfEvaluation`
  - `managerFeedback`
  - `scoreItems`
  - 薪资金额
- 目标地图导出仅包含：
  - `employeeName`
  - `departmentName`
  - `title`
  - `targetValue`
  - `currentValue`
  - `unit`
  - `weight`
  - `status`
  - `startDate`
  - `endDate`
  - `updateTime`
- 目标地图导出不包含：
  - 单条他人环评反馈全文
  - 无权限审批意见
  - 薪资金额

### 最小验收标准

#### 正常路径

1. `HR` 可导出 `feedback / pip / promotion / salary` 冻结允许范围内的数据。
2. 部门经理可导出 `feedback / pip / promotion` 冻结允许范围内的数据。
3. 导出文件格式固定为 `xlsx`。
4. 导出列必须与 [12-数据权限与脱敏规则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/12-数据权限与脱敏规则.md) 的字段矩阵完全一致。
5. 导出成功后必须生成审计记录。

#### 异常路径

1. 员工访问任一二期导出接口时被拒绝。
2. 部门经理访问 `salary/export` 时被拒绝。
3. 无数据时返回 `当前筛选条件下无可导出数据`。
4. 超过导出上限时返回 `导出结果超过上限，请缩小筛选范围后重试`。

#### 边界路径

1. 部门经理按部门树边界导出时，不能拿到跨部门数据。
2. `feedback/export` 只能导出摘要，不能导出 `records[]` 单条反馈。
3. `pip/export` 只能导出摘要，不能导出改进目标和跟进全文。
4. `promotion/export` 只能导出摘要，不能导出评审意见全文。
5. `salary/export` 只能导出非金额摘要，不能导出金额字段。

#### 权限与脱敏路径

1. 下载权限必须独立校验，不能因已有 `page / info` 权限自动放开。
2. 后端返回的导出文件列必须再次按角色裁剪，不能只依赖前端按钮隐藏。
3. 导出失败、审计失败、越权失败日志中不得泄露敏感字段原值。

### 最小测试矩阵

| 维度 | 必测点 |
| --- | --- |
| 正常路径 | 按冻结字段矩阵成功导出 `xlsx`，列头与行数正确 |
| 异常路径 | 无权限、无数据、超限三类错误语义正确 |
| 边界 | 经理部门树、摘要导出、金额不导出、全文不导出 |
| 权限 | `HR / 经理 / 员工` 三角色下载权限矩阵正确 |
| 脱敏 | 导出文件不出现绝对禁止字段 |
| 审计 | 成功与失败两类导出都生成审计留痕 |
