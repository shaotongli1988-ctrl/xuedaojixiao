### 后端测试现实约束

当前 `cool-admin-midway` 项目自带说明里已经明确：自动化路由模式下，不推荐把 `jest` 作为主测试路径。现阶段更适合的验证组合是：

   - ApiPost

1. 首期并不是“完全没有自动化测试”，而是自动化测试先收敛到现有仓库能力。

3. 前端首期自动化验证以 `type-check` 和 `build` 为主，不额外引入新的 E2E 框架。

## 测试目标

为首期绩效域模块提供最小可交付回归矩阵，保证下面 6 类高风险问题不会在首期上线前漏测：

1. 接口字段和页面字段不一致
2. 状态机流转错误
3. 权限显隐与后端鉴权不一致
4. 敏感数据越权返回
5. 驾驶舱统计口径错误
6. 关键动作执行后页面状态未刷新

## 最小回归矩阵

### 1. 绩效驾驶舱

#### 正常路径

- 能按默认周期加载驾驶舱数据
- KPI 卡片有值且字段完整
- 部门筛选后数据刷新
- 图表无空指针报错

#### 异常路径

- 无权限用户访问驾驶舱接口时被拒绝
- 无数据周期返回空态，不报错

#### 边界

- 部门无任何评估数据时，分布图返回空数组
- 周期内只有一条数据时，均分和分布仍可计算
- 首期接口返回中不包含跨模块保留指标字段，前端不依赖 `reservedMetrics`

### 2. 评估单主流程

#### 正常路径

- HR 或经理可以创建评估单
- 员工可查看本人评估单详情
- 草稿评估单可提交
- 已提交评估单可被部门经理审批通过
- 已提交评估单可被部门经理审批驳回

#### 异常路径

- 员工不能创建评估单
- 非草稿不能再次提交
- 非已提交状态不能执行审批
- HR 不能执行审批通过或驳回
- 不存在的评估单返回明确错误

#### 边界

- 周期重复但编号不同的评估单可区分
- 评分项为空时禁止提交或按设计规则拒绝
- 前端手工传入 `grade` 时，服务端不接受直接覆盖最终等级

### 3. 我的考核 / 已发起考核 / 待我审批三视图一致性

#### 正常路径

- 我的考核只展示当前登录人数据
- 已发起考核只展示自己发起或可管理范围数据
- 待我审批只展示 `submitted` 且当前人有审批权的数据
- 我的考核默认按 `updateTime desc` 排序
- 已发起考核默认按 `createTime desc` 排序
- 待我审批默认按 `submitTime asc` 排序

#### 异常路径

- 员工不能看到待我审批页面数据
- 经理不能越权看到非所辖部门考核
- HR 在待我审批页面不能执行审批按钮

#### 边界

- 同一评估单状态变化后，三个列表展示同步变化
- 同一条评估单可同时出现在“已发起考核”和“待我审批”中，但展示原因必须符合视角规则
- 经理可看到直接管理部门及其下级部门数据
- 经理不能看到跨部门兼任场景下未授权部门数据

### 4. 目标地图

#### 正常路径

- 可创建目标
- 可编辑目标
- 可更新目标进度
- 目标完成后状态变更正确

#### 异常路径

- 已完成目标不能继续更新进度
- 非本人或无权限用户不能修改目标

#### 边界

- 当前值等于目标值时，允许进入完成状态
- 当前值超过目标值时，按设计规则处理且不报错

### 5. 指标库

#### 正常路径

- HR 可新增、编辑、删除指标
- 指标列表按状态筛选有效

#### 异常路径

- 非 HR 访问新增或编辑接口被拒绝
- 重复指标编码被拒绝

### 5.1 导出能力

#### 正常路径

- `HR` 可执行绩效评估导出
- `HR` 和 `部门经理` 可执行目标地图导出

#### 异常路径

- `部门经理` 不可执行绩效评估导出
- `员工` 不可执行任何导出
- 首期访问 `薪资`、`环评`、`PIP`、`晋升` 导出能力时，应无入口或被拒绝

#### 敏感数据测试

- 绩效评估导出仅包含：
  - `code`
  - `employeeName`
  - `departmentName`
  - `periodType`
  - `periodValue`
  - `assessorName`
  - `status`
  - `targetCompletion`
  - `totalScore`
  - `grade`
  - `submitTime`
  - `approveTime`
- 绩效评估导出不包含：
  - `selfEvaluation`
  - `managerFeedback`
  - `scoreItems`
  - 薪资金额
- 目标地图导出仅包含：
  - `employeeName`
  - `departmentName`
  - `title`
  - `targetValue`
  - `currentValue`
  - `unit`
  - `weight`
  - `status`
  - `startDate`
  - `endDate`
  - `updateTime`
- 目标地图导出不包含：
  - 单条他人环评反馈全文
  - 无权限审批意见
  - 薪资金额

### 6. 360 环评

#### 正常路径

- 可发起环评任务
- 评价人可提交反馈
- 可查看汇总结果
- `HR` 和对应部门经理可查看单条反馈内容

#### 异常路径

- 已关闭任务不可再提交反馈
- 非任务评价人不可提交反馈
- 同一评价人不可重复提交
- 截止后不可补交

#### 边界

- 只收到一条反馈时，汇总结果仍可展示
- 所有反馈未完成时，不允许标记为完成汇总
- 员工本人只能看汇总结果，不能看单条他人反馈

### 7. PIP

#### 正常路径

- 可创建 PIP
- 可启动 PIP
- 可提交跟进记录
- 可标记完成或关闭
- 从评估详情创建时，能自动带入 `assessmentId`
- 独立创建时，能保存原因说明

#### 异常路径

- `draft` 状态未启动前不能直接完成
- 已完成或已关闭的 PIP 不允许新增跟进
- 非负责人不能写跟进

### 8. 晋升管理

#### 正常路径

- 可发起晋升单
- 草稿晋升单可提交进入 `reviewing`
- 评审人可提交评审结论
- 状态从 `reviewing` 正常进入 `approved` 或 `rejected`
- 从评估详情创建时，能自动带入 `assessmentId`
- 独立创建时，能保存原因说明

#### 异常路径

- 非评审人不能执行评审
- 非 `reviewing` 状态不能评审

### 9. 薪资管理

#### 正常路径

- HR 可查看薪资列表和详情
- HR 可新增薪资调整记录
- HR 可录入真实金额字段
- `draft` 状态薪资可编辑
- `draft` 状态薪资可确认
- `confirmed` 状态薪资可归档

#### 异常路径

- 经理访问薪资详情被拒绝或返回脱敏结果
- 员工不能访问后台薪资管理页面
- `confirmed` 状态薪资不能直接修改金额
- 非 HR 调用 `changeAdd` 被拒绝

#### 敏感数据测试

- 无权限角色不返回薪资字段
- 普通错误提示不带薪资金额
- 日志不输出完整薪资明细
- 首期不存在薪资导出入口

## 状态流转回归矩阵

### 评估单

- `draft -> submitted` 通过
- `submitted -> approved` 通过
- `submitted -> rejected` 通过
- `approved -> submitted` 拒绝

### 评分与等级

- `totalScore` 按 `sum(score * weight) / sum(weight)` 计算正确
- 目标完成率只参与一次评分，不重复叠加
- `90` 分进入 `S`
- `80` 分进入 `A`
- `70` 分进入 `B`
- `< 70` 进入 `C`
- 不允许手工覆盖 `grade`

### 目标

- `draft -> in-progress` 通过
- `in-progress -> completed` 通过
- `completed -> in-progress` 拒绝

### 环评

- `draft -> running` 通过
- `running -> closed` 通过
- `closed -> submitted` 拒绝

### PIP

- `draft -> active` 通过
- `active -> completed` 通过
- `completed -> active` 拒绝

### 晋升

- `draft -> reviewing` 通过
- `reviewing -> approved` 通过
- `approved -> rejected` 拒绝

### 薪资

- `draft -> confirmed` 通过
- `confirmed -> archived` 通过
- `archived -> confirmed` 拒绝
