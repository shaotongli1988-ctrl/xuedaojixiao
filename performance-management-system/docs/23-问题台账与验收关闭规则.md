# 问题台账与验收关闭规则

## 文档职责

这份文档是“问题台账和验收关闭规则”的唯一事实源。

它只负责定义：

1. 问题分级
2. 问题台账最小字段
3. 问题状态流转
4. 复验规则
5. 验收关闭规则

## 不负责内容

这份文档不负责定义：

- 需求本身
- 详细测试案例
- 发布回滚步骤

这些内容分别以 `01`、`09`、`15` 文档为准。

## 问题分级

| 级别 | 定义 |
| --- | --- |
| `P0` | 越权、敏感数据暴露、数据写错、主路径中断、无回滚能力 |
| `P1` | 关键行为与事实源不一致、关键验证缺失、联调或验收无法继续 |
| `P2` | 不影响正确性和交付安全的体验、文案、维护性问题 |

## 台账最小字段

每条问题至少记录：

1. 问题编号
2. 标题
3. 来源阶段
4. 严重级别
5. 所属模块
6. 发现人
7. 负责人
8. 当前状态
9. 复现步骤
10. 预期结果
11. 实际结果
12. 关联文档或代码范围
13. 计划修复时间
14. 复验结果

## 状态集合

| 状态 | 含义 |
| --- | --- |
| `open` | 新发现，未开始处理 |
| `in-progress` | 正在修复 |
| `resolved` | 已修复，待复验 |
| `reopened` | 复验未通过，重新打开 |
| `closed` | 复验通过，正式关闭 |
| `waived` | 经正式批准延期处理 |

## 状态流转规则

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `open` | 指派修复 | `in-progress` |
| `in-progress` | 修复完成 | `resolved` |
| `resolved` | 复验通过 | `closed` |
| `resolved` | 复验失败 | `reopened` |
| `reopened` | 再次修复 | `in-progress` |
| `open` / `in-progress` / `resolved` | 正式批准延期 | `waived` |

补充规则：

1. `waived` 只允许 `P2` 使用。
2. `P0 / P1` 不允许以 `waived` 方式关闭当前阶段。
3. `closed` 后如果再次发现同类问题，开新记录并关联原编号。

## 复验规则

1. 修复人不能作为唯一复验人。
2. 复验必须记录：
   - 复验时间
   - 复验人
   - 复验结果
   - 证据说明
3. 涉及权限、状态机、脱敏、导出的修复，必须做针对性回归。

## 验收关闭规则

### 单问题关闭

满足以下条件才能关闭：

1. 已复现或已确认问题存在
2. 已完成修复
3. 已完成复验
4. 关联文档已回写或确认无需回写

### 阶段验收关闭

满足以下条件才能标记当前阶段验收完成：

1. 当前阶段 `P0 / P1` 全部关闭
2. 所有 `P2` 已明确 owner 和计划时间
3. 遗留项不影响当前阶段门禁
4. 验收结论已记录

## 例外记录

如果 `P2` 允许延期，必须记录：

1. 批准人
2. 延期原因
3. 风险说明
4. owner
5. 计划关闭时间
6. 临时缓解措施

## 与其他文档的关系

1. 问题影响需求定义，回写 `01`、`04`、`05`、`06`、`07`、`12`
2. 问题影响实施顺序或阶段判断，回写 `19`、`20`
3. 问题影响交付边界，回写 `14`、`15`
4. 问题关闭或例外记录后，更新 `13`
