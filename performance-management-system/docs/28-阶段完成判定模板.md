# 阶段完成判定模板

## 文档职责

这份文档是“阶段状态如何判定”的唯一事实源。

它只负责定义：

1. 什么叫代码完成
2. 什么叫联调完成
3. 什么叫可提测
4. 什么叫可进入下一阶段

## 不负责内容

这份文档不负责定义：

- 模块具体实现细节
- 问题台账全文
- 测试用例全文

## 状态定义

### 1. 代码完成

只有同时满足下面条件，才能把某个模块判定为“代码完成”：

1. 代码已落到现有技术基座
2. 对应任务卡已创建并更新
3. 当前范围内的自动化验证已执行
4. 文档事实源已同步
5. 没有未记录的行为变化

“代码完成”不等于“联调完成”。

### 2. 联调完成

只有同时满足下面条件，才能把模块判定为“联调完成”：

1. 真实联调账号可登录
2. 真实联调主接口可访问
3. 权限与数据范围通过
4. 当前阶段 smoke 通过
5. 联调结果已回写任务卡

只要 smoke 未通过，联调仍视为未完成。

### 3. 可提测

只有同时满足下面条件，才允许进入提测：

1. 代码完成
2. 联调完成
3. 当前无未关闭 `P0 / P1`
4. 已知限制已写入文档
5. 回滚和交付前提已明确

### 4. 可进入下一阶段

只有同时满足下面条件，才允许进入下一阶段：

1. 当前阶段完成判定已满足
2. 当前阶段证据已回写
3. 当前阶段遗留项已分级
4. 下一阶段入口条件已满足

## 阶段判定模板

每次要推动状态时，必须至少回答下面 4 组问题：

### A. 当前想改成什么状态

1. 从什么状态改到什么状态
2. 这次推进针对哪个任务卡

### B. 证据是什么

1. 跑了哪些命令
2. 哪些通过
3. 哪些没跑
4. 剩余风险是什么

### C. 阻塞是否解除

1. 当前是否还有 `P0 / P1`
2. 若有，为什么还想推进

### D. 回写是否完成

1. 任务卡是否回写
2. 版本记录是否回写
3. 问题台账是否回写

## 明确禁止的状态误判

下面这些情况都不允许被判定为完成：

1. 单测过了，但真实联调没做
2. 登录通了，但受保护接口没通
3. 前端页面能打开，但权限和数据范围没验证
4. 代码改完了，但任务卡没回写
5. 文档修正了，但运行时问题没修复
6. 某个窗口说完成了，但总任务卡没有最终证据

## 推荐回写格式

任务卡或阶段记录至少应包含：

1. 当前状态
2. 执行命令
3. 执行结果
4. 阻塞项
5. 下一步
