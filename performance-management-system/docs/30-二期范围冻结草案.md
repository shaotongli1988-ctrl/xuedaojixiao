# 二期范围正式冻结版

## 文档职责

这份文档用于冻结“二期总范围”，只做开发前准备，不授权直接进入设计实现。

它只负责：

1. 收口当前允许进入二期阶段 0 评审的正式范围
2. 说明为什么这些能力放到二期
3. 说明每项候选的业务价值、依赖和缺失事实源
4. 给出正式优先级、准入条件和禁止开工条件

它不负责：

1. 定义二期正式需求全文
2. 定义二期 API、表结构、状态机或权限细节
3. 把二期候选提前并入首期阶段 4 验收或阶段 5 交付
4. 把“建议优先级”直接写成排期承诺或开发开始指令

  - 二期仍只允许做“冻结、澄清、验收口径、任务拆分”
  - 当前技术边界固定为 `cool-admin-vue + cool-admin-midway + cool-uni`
- 当前不允许的动作：
  - 直接进入二期实现
  - 以聊天结论替代冻结包
  - 把未补齐事实源的候选项推进到设计或开发

## 二期总目标

1. 冻结二期唯一候选范围，确保不再混入首期已关闭事项。
2. 明确二期优先级、进入条件、不得开工条件和首开项。
3. 把每个候选项推进到“可启动开发前评审”的正式冻结包，而不是停留在口头草案。

## 二期非目标

1. 不修改首期已关闭的功能、权限、状态机和交付结论。
2. 不开发前后端业务代码、不调整共享实现、不新增脚手架或基础设施。
3. 不把任何候选项写成“已决定开发”或“已完成设计”。
4. 不在当前阶段补二期 API、表结构、状态机、权限正文。

## 冻结原则

1. 当前二期候选只允许来自首期已明确“不纳入范围”的事项
2. 当前技术边界仍固定为 `cool-admin-vue + cool-admin-midway + cool-uni`
3. 任一候选在补齐事实源前，不得直接进入设计或开发
4. 当前文档给出的是“正式冻结优先级”，不等同于排期承诺

## 进入条件

1. 首期阶段 4 / 阶段 5 已闭环，且其结论不再回退。
2. 二期候选仅来自首期明确“不纳入范围”的事项。
3. 当前冻结结论、阻塞项、证据和下一步已回写仓库文档 / 任务卡。

## 不得开工条件

1. 任一候选项缺少最小事实源清单时，不得进入设计或开发。
2. 任一候选项若会改首期共享实现、首期状态机或首期已关闭结论，必须先单独冻结再评审。
3. 任一候选项若只能依赖聊天说明、截图或口头共识，不得推进。
4. 未建任务卡、未回写版本记录、未完成阶段 0 守卫留痕时，不得视为冻结完成。

### 4. 自动建议引擎

- 为什么放二期：
  - 首期已明确 PIP / 晋升只支持手工创建，不做系统自动建议
- 业务价值：
  - 降低 HR 和经理人工筛查成本
  - 把评分结果转成可执行动作建议
- 前置依赖：
  - 绩效阈值和建议规则
  - 建议是否可覆盖、忽略、驳回
  - 建议记录是否需要审计
- 当前已能冻结的边界：
  - 当前只允许讨论“建议能力”，不允许顺带扩成自动审批流
  - 首批建议对象固定为 `PIP 建议` 与 `晋升建议`，不扩到薪资、环评、目标、跨模块驾驶舱或外部数据域
  - 当前只允许围绕现有评估结果域的稳定结果字段讨论，不允许从目标、环评、薪资、驾驶舱或外部系统直接触发
  - 当前不能改写首期已关闭的 `PIP / 晋升` 手工创建主链
  - 建议只允许作为“推荐记录”，不允许自动创建正式 `PIP / 晋升` 单据
  - 首批触发口径固定为：
    1. `assessment.status = approved` 才允许触发
    2. `assessment.grade = C` 或 `assessment.totalScore < 70` 触发 `PIP 建议`
    3. `assessment.grade = S / A` 或 `assessment.totalScore >= 80` 触发 `晋升建议`
    4. `assessment.grade = B` 默认不触发建议
  - 同一员工、同一评估单下已存在同类未结束人工处理结果时，不允许重复触发同类建议
- 不做会有什么影响：
  - PIP / 晋升仍需依赖人工判断和发起
  - 运营效率提升有限
- 当前冻结补齐结果：
  - `04` 已补齐 `suggestion` 资源接口、动作语义、错误语义和与既有 `PIP / 晋升` 主链的关系
  - `05` 已补齐建议状态、合法 / 非法流转和人工确认链路
  - `06` 已补齐 `HR / 经理 / 员工` 的查看和处理权限矩阵
  - `09` 与 `24` 已补齐测试矩阵和开发拆分顺序
  - `12` 已补齐建议摘要字段、角色可见粒度与字段级脱敏边界

  - 当前主题 4 已形成单一一致的阶段 0 事实源，后续实现只能按已冻结的接口、状态、权限、脱敏和测试矩阵推进

  - `自动建议引擎已完成阶段0冻结，可进入开发`
