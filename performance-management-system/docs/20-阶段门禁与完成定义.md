# 阶段门禁与完成定义

## 文档职责

这份文档是“项目阶段门禁和完成定义”的唯一事实源。

它只负责定义：

1. 各阶段的进入条件
2. 各阶段的退出条件
3. 阻塞进入下一阶段的 `P0 / P1` 项
4. 每个阶段必须具备的交付证据

## 不负责内容

这份文档不负责定义：

- 模块字段和接口细节
- 具体页面结构
- 详细测试案例

这些内容分别以 `03`、`04`、`18`、`09` 文档为准。

## 总规则

1. 任何阶段存在未关闭的 `P0` 或 `P1` 阻塞项，不进入下一阶段。
2. 如果门禁证据不完整，视为本阶段未完成。
3. 阶段通过后仍可补 `P2`，但必须记录 owner、完成时点和清理动作。
4. 阶段通过判断以本文件为准，不以口头确认或聊天记录为准。
5. 涉及行为变化的工作项，必须满足 [24-自动化测试策略与脚本规划.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/24-自动化测试策略与脚本规划.md) 定义的自动化验证门槛，否则视为 `P1` 阻塞。
6. 命中菜单、路由、`viewPath`、权限键、高价值文档回写或自动化脚本边界的工作项，必须通过仓库一致性守卫；守卫失败按 `P1` 处理。

## 阶段 0：开发前冻结

### 进入条件

1. 已确认首期只使用 `cool-admin-vue + cool-admin-midway`。
2. 已确认 `cool-uni` 不进入首期实现。
3. 项目文档入口和唯一事实源目录已建立。

### 退出条件

1. `01`、`03`、`04`、`05`、`06`、`07`、`08`、`10`、`12`、`18` 文档可支撑首个模块开发。
2. 首期角色、状态、权限、菜单、字段、导出边界已冻结。
3. 首个模块的代码目录落点已明确。

### 必备证据

1. [README.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/README.md)
2. [17-项目执行流程（精简版）.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/17-项目执行流程（精简版）.md)
3. [19-模块开发任务拆分.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/19-模块开发任务拆分.md)
4. [25-项目最高准则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/25-项目最高准则.md)

### 阻塞项

- `P0`：首期业务流程、权限或状态机仍有歧义
- `P1`：首个模块缺少 API、数据库或权限事实源
- `P1`：项目最高准则或自动化门槛未落版，仍试图直接进入编码

## 阶段 1：模块开发中

### 进入条件

1. 当前模块在 [19-模块开发任务拆分.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/19-模块开发任务拆分.md) 中已有明确任务边界。
2. 已创建任务卡并指定负责人。
3. 当前模块依赖的前置模块已达到最小完成定义。

### 退出条件

1. 当前模块功能已达到最小完成定义。
2. 正常、异常、边界路径至少各完成一次验证。
3. 受影响唯一事实源文档已回写。

### 必备证据

1. 当前模块任务卡
2. 最小验证记录
3. 受影响文档回写记录
4. 满足 [24-自动化测试策略与脚本规划.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/24-自动化测试策略与脚本规划.md) 定义的自动化验证结果

### 阻塞项

- `P0`：权限越权、脱敏失效、状态流转错误、数据写错
- `P1`：模块改了但文档、菜单、权限、接口未同步
- `P1`：存在行为变化但没有自动化验证结果
- `P1`：命中一致性守卫范围但未通过目录命名、菜单 / 路由 / `viewPath`、权限键或高价值文档回写校验

## 阶段 2：模块联调

### 进入条件

1. 当前模块前后端实现已具备最小可运行路径。
2. 联调账号、角色和种子数据满足 [14-交付说明.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/14-交付说明.md) 口径。
3. 菜单、按钮权限和接口权限已导入或可模拟。

### 退出条件

1. 主路径联调通过。
2. 关键权限路径联调通过。
3. 联调发现的问题已进入问题台账。

### 必备证据

1. 联调账号清单
2. 联调最小数据集
3. 联调问题记录

### 阻塞项

- `P0`：主路径不通、权限绕过、敏感数据暴露
- `P1`：联调环境无法复现实测口径、问题只在聊天中流转未入台账

## 阶段 3：提测前收口

### 进入条件

1. 本次提测范围已冻结。
2. 涉及模块的最小验证已完成。
3. 问题台账中的 `P0 / P1` 已关闭或明确阻塞提测。

### 退出条件

1. 提测版本范围明确。
2. 测试入口、账号、步骤、预期结果清楚。
3. 已知未开放能力和限制已记录。

### 必备证据

1. [09-测试清单.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/09-测试清单.md)
2. [14-交付说明.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/14-交付说明.md)
3. 问题台账最新状态
4. 自动化测试执行结果

### 阻塞项

- `P0`：提测内容和文档、代码、权限不一致
- `P1`：测试账号、数据、入口不完整，导致无法验收
- `P1`：自动化测试应执行未执行，或执行结果未留痕
- `P1`：命中一致性守卫范围但未留通过结果或存在未关闭漂移项

## 阶段 4：业务验收

### 进入条件

1. 业务验收主路径已在交付说明中明确。
2. 验收范围内的已知问题已分级。
3. 需求冻结状态明确，临时新增项不混入当前验收。

### 退出条件

1. 验收结论明确为通过、部分通过或驳回。
2. 遗留问题都已进入问题台账并指定 owner。
3. 需求变更与缺陷修复已区分记录。

### 必备证据

1. 验收记录
2. 问题台账
3. 版本影响范围说明

### 阻塞项

- `P0`：关键主路径未通过仍试图强行验收关闭
- `P1`：验收意见没有落台账、没有 owner、没有时限

## 阶段 5：上线前交付

### 进入条件

1. 文档、代码、菜单、权限、测试结果一致。
2. 交付说明和回滚说明可执行。
3. 高风险问题已关闭或有正式例外记录。

### 退出条件

1. 交付包可用。
2. 回滚路径明确。
3. 发布后若出现问题，有可执行的排查入口和责任边界。

### 必备证据

1. [14-交付说明.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/14-交付说明.md)
2. [15-回滚说明.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/15-回滚说明.md)
3. 最新问题台账
4. 最终版本记录

### 阻塞项

- `P0`：无回滚路径、无数据恢复路径、无权限回退路径
- `P1`：交付说明缺少入口、初始化、角色或限制说明

## 例外处理

1. 只有 `P2` 可以带条件放行。
2. 带条件放行必须记录：
   - 问题内容
   - 风险等级
   - owner
   - 计划完成时间
   - 清理动作
3. `P0 / P1` 不允许口头放行。
