# 项目最高准则

## 文档职责

这份文档是 `performance-management-system` 的项目级最高准则。

它只负责定义：

1. 项目不可绕过的总规则
2. 防止范围、流程、实现和技术栈漂移的红线
3. 进入开发、推进阶段和关闭任务时的最高优先级判断规则

## 不负责内容

这份文档不负责定义：

- 具体业务需求全文
- 具体字段、接口、表结构和状态流转细节
- 具体页面原型和交互稿

这些内容分别以 `01`、`03`、`04`、`05`、`18` 文档为准。

## 项目最高准则

1. 本项目唯一事实源入口只有 [README.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/README.md)，所有开发、评审、联调、交付都从这里进入。
2. 项目规则以 `performance-management-system/docs/` 为准；聊天记录、口头说明、临时截图、旧文档、历史页面都不能直接覆盖这里的结论。
3. 首期技术边界固定为 `cool-admin-vue + cool-admin-midway`，`cool-uni` 明确放到二期；不得为首期需求新增第二套前后端工程、测试框架或流程体系。
4. 代码实现必须复用现有 Cool 架构能力，包括动态菜单导入、`viewPath` 路由映射、现有模块加载方式和现有权限链路，不得自行发明并行机制。
5. 模块开发必须按 [19-模块开发任务拆分.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/19-模块开发任务拆分.md) 的顺序推进；前一模块存在未关闭 `P0 / P1` 时，不进入下一模块编码。
6. 任何模块进入编码前，必须先建任务卡，并按 [21-任务卡与状态推进规则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/21-任务卡与状态推进规则.md) 维护状态、结论、验证和回写范围。
7. 任何需求、契约、状态、权限、菜单、导出、脱敏、流程规则变化，必须先更新对应唯一事实源，再继续改代码；不允许只改实现不改文档。
8. 任何行为变更都必须具备自动化验证证据，最少满足 [24-自动化测试策略与脚本规划.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/24-自动化测试策略与脚本规划.md) 的项目门槛；没有自动化验证证据，不得标记完成、提测或进入下一阶段。
9. `P0 / P1` 为硬阻塞，不能口头放行；只有 `P2` 可以带 owner、截止时间和清理动作后条件放行，具体以 [20-阶段门禁与完成定义.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/20-阶段门禁与完成定义.md) 为准。
10. 任何实质性规则调整都必须同步回写 [13-版本与变更记录.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/13-版本与变更记录.md)，否则不视为正式变更。
11. 任何阶段联调必须满足 [26-可执行联调门禁与Smoke规则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/26-可执行联调门禁与Smoke规则.md)；没有 smoke 通过记录，不得视为联调完成。
12. 任何共享基础层改动必须满足 [27-共享基础层变更治理规则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/27-共享基础层变更治理规则.md)；只修局部接口不验证登录后真实路径，视为未完成。
13. 任何状态推进都必须满足 [28-阶段完成判定模板.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/28-阶段完成判定模板.md)；窗口完成不等于阶段完成。

## 项目级红线

1. 不允许把二期内容挤入首期实现。
2. 不允许新增脱离现有架构的新工程、新框架、新菜单机制或新权限机制。
3. 不允许未建任务卡直接开始模块编码。
4. 不允许未更新唯一事实源就修改行为、契约、状态、权限或流程。
5. 不允许把手工验证、聊天确认或“本地看起来可以”替代自动化验证证据。
6. 不允许跳过 `P0 / P1` 阻塞项继续推进。
