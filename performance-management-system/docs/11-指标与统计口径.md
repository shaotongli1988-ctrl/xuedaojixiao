# 指标与统计口径

## 文档职责

这份文档是“驾驶舱和统计指标口径”的唯一事实源。

它只负责定义：

1. 指标名称
2. 计算公式
3. 时间范围
4. 数据来源
5. 图表维度
6. 刷新频率

## 不负责内容

这份文档不负责定义：

- 图表组件样式
- 页面按钮权限
- 表结构字段类型

## 首期范围原则

首期驾驶舱只统计绩效域自身能直接产出的数据，不做跨招聘、培训、会议模块的实时聚合。原因如下：

1. 当前首期开发范围只包含绩效域。
2. 现有技术架构中还没有这几个模块的统一统计接口。
3. 强行在首期引入跨模块指标，会让文档和实现不一致。

因此 PRC 文档中的以下指标在首期处理为“保留扩展位，不纳入实际交付口径”：

- 招聘达成率
- 内训通关率
- 会议效能看板

确认结论：

1. 首期驾驶舱只交付绩效域内 6 个真实指标。
2. 上述跨招聘、培训、会议模块指标不在首期页面中展示。
3. 相关能力统一延后到二期评估和实现。

## 首期正式纳入指标

### 总分计算基线

- 评估单 `totalScore` 统一按 `sum(score * weight) / sum(weight)` 计算
- 目标完成率在首期只作为一个普通评分项参与加权
- `grade` 按 `S / A / B / C` 分数区间由服务端自动计算
- 首期不允许手工覆盖等级

### 1. 全员绩效均分

- 指标名称：全员绩效均分
- 公式：`sum(approved 状态评估单 totalScore) / count(approved 状态评估单)`
- 时间范围：按筛选周期
- 数据来源：`performance_assessment`
- 维度：按时间筛选，可加部门筛选
- 刷新频率：页面进入时实时计算

### 2. 待审批数量

- 指标名称：待审批数量
- 公式：`count(status = submitted 的评估单)`
- 时间范围：当前筛选周期
- 数据来源：`performance_assessment`
- 维度：按当前用户权限范围
- 刷新频率：页面进入时实时计算

### 3. 目标完成率

- 指标名称：目标完成率
- 公式：`count(status = completed 的目标) / count(全部目标)`
- 时间范围：当前筛选周期
- 数据来源：`performance_goal`
- 维度：按部门、员工、周期
- 刷新频率：页面进入时实时计算
- 周期映射补充：由于 `performance_goal` 首期没有独立 `periodValue` 字段，驾驶舱筛选的 `month / quarter / year` 统一先映射为日期区间，再按 `goal.startDate <= periodEnd` 且 `goal.endDate >= periodStart` 的时间重叠规则纳入口径。

### 4. 五环节进度

- 指标名称：绩效全景五环节进度
- 口径定义：
  1. 指标库配置完成度：`count(status = 1 的指标) / count(全部指标)`
  2. 评估单创建完成度
  3. 自评提交完成度
  4. 审批完成度
  5. 结果归档完成度：`count(status = archived 的薪资结果) / count(全部薪资结果)`
- 公式：每一环节分别按“已完成数 / 应完成数”
- 数据来源：
  - `performance_indicator`
  - `performance_assessment`
  - `performance_salary`
- 维度：仅按周期
- 刷新频率：页面进入时实时计算

### 5. 部门绩效分布

- 指标名称：部门绩效分布
- 公式：按部门分组，统计 `approved` 评估单平均分
- 数据来源：`performance_assessment`
- 维度：部门
- 刷新频率：页面进入时实时计算

### 6. 绩效等级分布

- 指标名称：绩效等级分布
- 公式：按 `grade` 分组统计数量
- 等级集合：`S / A / B / C`
- 数据来源：`performance_assessment`
- 维度：等级
- 刷新频率：页面进入时实时计算

## 保留扩展位指标

### 招聘达成率

- 首期状态：不实现
- 原因：首期不开发招聘模块接口和表结构
- 处理方式：首期前端不展示，二期再评估是否纳入驾驶舱

### 内训通关率

- 首期状态：不实现
- 原因：首期不开发培训模块接口和表结构
- 处理方式：首期前端不展示，二期再评估是否纳入驾驶舱

### 会议效能看板

- 首期状态：不实现
- 原因：首期不开发会议模块接口和表结构
- 处理方式：首期前端不展示，二期再评估是否纳入驾驶舱

## 图表口径说明

| 图表 | 类型 | 数据来源 |
| --- | --- | --- |
| KPI 卡片 | 数字卡片 | `dashboard/summary` 聚合接口 |
| 五环节进度 | 柱状图或进度条 | `dashboard/summary.stageProgress` |
| 部门绩效分布 | 柱状图 | `dashboard/summary.departmentDistribution` |
| 绩效等级分布 | 饼图 | `dashboard/summary.gradeDistribution` |

## 实现约束

1. 所有指标都由后端聚合接口统一计算。
2. 前端不对多个资源结果做二次统计。
3. 统计口径一旦变更，必须同步更新本文件和 API 文档。
