# 字段字典与枚举

## 文档职责

这份文档是“公共字段定义和枚举”的唯一事实源。

它只负责定义：

1. 状态枚举
2. 等级枚举
3. 指标类型
4. 审批结果
5. 敏感字段标识
6. 页面下拉和字典项来源

## 不负责内容

这份文档不负责定义：

- 表之间关系
- 状态流转顺序
- 接口路径

## 绩效等级

### 等级集合

首期绩效等级固定为 4 档：

- `S`
- `A`
- `B`
- `C`

### 分数区间

| 等级 | 分数区间 | 说明 |
| --- | --- | --- |
| `S` | `>= 90` | 最优档 |
| `A` | `>= 80 且 < 90` | 良好档 |
| `B` | `>= 70 且 < 80` | 达标档 |
| `C` | `< 70` | 待改进档 |

说明：

1. 首期等级名称和分数区间已经确认。
2. 首期等级不允许手工覆盖，必须由服务端按 `totalScore` 自动计算。

## 总分公式

- `totalScore = sum(score * weight) / sum(weight)`
- 目标完成率作为一个普通评分项参与上述加权公式
- 首期不支持在前端直接传入最终 `grade`

## 评估状态

| 值 | 标签 | 说明 |
| --- | --- | --- |
| `draft` | 草稿 | 员工可保存自评，发起人可编辑 |
| `submitted` | 已提交 | 等待部门经理审批 |
| `approved` | 已审批通过 | 评估结果生效 |
| `rejected` | 已驳回 | 退回员工重新编辑 |

## 目标状态

| 值 | 标签 | 说明 |
| --- | --- | --- |
| `draft` | 草稿 | 未正式发布 |
| `in-progress` | 进行中 | 已发布并持续更新进度 |
| `completed` | 已完成 | 目标完成 |
| `cancelled` | 已取消 | 首期不支持恢复 |

## 指标类型

| 值 | 标签 | 说明 |
| --- | --- | --- |
| `assessment` | 考核指标 | 用于评估单评分项 |
| `goal` | 目标指标 | 用于目标地图关联指标 |
| `feedback` | 环评指标 | 用于环评维度或汇总分类 |

## 指标适用范围

| 值 | 标签 | 说明 |
| --- | --- | --- |
| `all` | 全员 | 全部角色或部门适用 |
| `department` | 部门 | 指定部门范围适用 |
| `employee` | 员工/岗位 | 指定岗位或员工类型适用 |

## 指标启停状态

| 值 | 标签 | 说明 |
| --- | --- | --- |
| `1` | 启用 | 可被新建业务单据引用 |
| `0` | 禁用 | 历史可读，不再用于新建 |

## 环评关系类型

首期固定支持以下关系类型：

- `leader`
- `peer`
- `subordinate`
- `collaborator`

说明：

1. 关系类型由发起任务时显式指定。
2. 首期不做匿名关系映射。

## 脱敏字段标记

| 业务对象 | 字段 | 脱敏级别 | 说明 |
| --- | --- | --- | --- |
| `assessment` | `scoreItems` | `L2` | 评分明细，需按角色裁剪 |
| `assessment` | `managerFeedback` | `L2` | 审批评语，对非授权人不可见 |
| `feedback` | `content` | `L2` | 单条他人环评内容，员工本人不可见 |
| `feedback` | `records` | `L2` | 汇总中的单条反馈集合，仅 HR/经理可见 |
| `promotion` | `comment` | `L2` | 晋升评审意见，按角色裁剪 |
| `salary` | `baseSalary` | `L3` | 高敏感金额字段，仅 HR 可见 |
| `salary` | `performanceBonus` | `L3` | 高敏感金额字段，仅 HR 可见 |
| `salary` | `adjustAmount` | `L3` | 高敏感金额字段，仅 HR 可见 |
| `salary` | `finalAmount` | `L3` | 高敏感金额字段，仅 HR 可见 |

说明：

1. `L2` 表示业务敏感字段，页面显示和导出都要按角色裁剪。
2. `L3` 表示高敏感金额字段，首期后台只允许 `HR` 查看和维护。
3. 具体脱敏策略和前后端职责以 [12-数据权限与脱敏规则.md](/Users/shaotongli/Documents/xuedao/performance-management-system/docs/12-数据权限与脱敏规则.md) 为准。
