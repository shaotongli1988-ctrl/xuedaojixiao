# 状态机与流转规则

## 文档职责

这份文档是“业务状态流转”的唯一事实源。

它只负责定义：

1. 每个业务对象的状态集合
2. 合法流转路径
3. 非法流转处理方式
4. 页面按钮和后端动作的对应关系

## 不负责内容

这份文档不负责定义：

- 表结构字段类型
- 接口字段格式
- 菜单可见性

## 总体规则

1. 状态流转以服务端为唯一事实源，前端只负责显隐和禁用。
2. 同一动作只能由一个明确状态触发，不能跨多个状态模糊执行。
3. 非法流转必须返回明确错误，不允许静默成功。

## 1. 评估单状态机

### 流程角色基线

- 发起人：`HR` 或 `部门经理`
- 填写人：`员工`
- 审批人：`部门经理`
- 归档和全局查看：`HR`

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿，可编辑 |
| `submitted` | 已提交，等待审批 |
| `approved` | 已审批通过 |
| `rejected` | 已驳回，允许回到编辑态 |

### 合法流转

| 当前状态 | 动作 | 下一状态 | 执行人 |
| --- | --- | --- | --- |
| `draft` | 新建 / 保存 | `draft` | HR / 部门经理 / 员工自评场景 |
| `draft` | 提交 | `submitted` | 员工 |
| `submitted` | 审批通过 | `approved` | 部门经理 |
| `submitted` | 审批驳回 | `rejected` | 部门经理 |
| `rejected` | 重新编辑保存 | `draft` | 员工 |

### 非法流转

- `approved` 不允许再提交或再编辑。
- `submitted` 不允许直接删除。
- `rejected` 不允许直接审批通过，必须先回到 `draft` 再重新提交。
- `HR` 首期不允许执行审批通过或审批驳回。
- 员工不允许新建评估单。

### 页面按钮映射

| 页面 | 状态 | 可见按钮 |
| --- | --- | --- |
| 我的考核 | `draft` | 保存、提交 |
| 我的考核 | `submitted` | 查看 |
| 已发起考核 | `draft` | 编辑、删除、查看 |
| 已发起考核 | `submitted` | 查看 |
| 待我审批 | `submitted` | 通过、驳回、查看 |

补充说明：

1. 首期评估单的唯一提交入口固定在“我的考核”，由员工执行。
2. `已发起考核` 页面不提供发起人代提交按钮。
3. 审批权固定只属于部门经理。

## 2. 目标状态机

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿 |
| `in-progress` | 进行中 |
| `completed` | 已完成 |
| `cancelled` | 已取消 |

### 合法流转

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `draft` | 首次更新进度 | `in-progress` |
| `in-progress` | 更新进度 | `in-progress` |
| `in-progress` | 达成目标值 | `completed` |
| `draft` | 取消 | `cancelled` |
| `in-progress` | 取消 | `cancelled` |

### 非法流转

- `completed` 不允许再更新进度。
- `cancelled` 不允许再恢复成进行中，首期需要重新建目标。
- 首期没有独立“发布目标 / 完成目标 / 取消目标”动作接口时，服务端通过 `progressUpdate` 自动承接 `draft -> in-progress` 与 `in-progress -> completed`。

## 3. 360 环评任务状态机

### 角色与可见范围基线

- 发起人：`HR` 或 `部门经理`
- 评价人：发起任务时显式选择
- 被评价员工：只能查看汇总结果
- `HR` 和对应 `部门经理`：可查看汇总和单条反馈

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 未发起 |
| `running` | 已发起，等待评价 |
| `closed` | 已关闭 |

### 环评记录状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 未提交 |
| `submitted` | 已提交 |

### 合法流转

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `draft` | 发起任务 | `running` |
| `running` | 提交反馈 | `running` |
| `running` | 全部完成或人工关闭 | `closed` |

### 非法流转

- `closed` 后不允许再新增反馈。
- `draft` 状态不允许查看汇总结果。
- 超过截止时间默认不允许补交。
- 同一评价人不允许对同一任务重复提交。

## 4. PIP 状态机

### 来源规则基线

- 支持从评估详情发起，自动带入 `assessmentId`
- 支持脱离评估单独立创建
- 首期不做自动建议触发

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿 |
| `active` | 进行中 |
| `completed` | 完成 |
| `closed` | 关闭 |

### 合法流转

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `draft` | 启动 PIP | `active` |
| `active` | 提交跟进 | `active` |
| `active` | 标记完成 | `completed` |
| `active` | 关闭 | `closed` |

### 非法流转

- `completed` 和 `closed` 不允许继续写跟进记录。
- `draft` 不允许直接完成。

## 5. 晋升状态机

### 来源规则基线

- 支持从评估详情发起，自动带入 `assessmentId`
- 支持脱离评估单独立创建
- 首期不做自动建议触发

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿 |
| `reviewing` | 评审中 |
| `approved` | 通过 |
| `rejected` | 驳回 |

### 合法流转

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `draft` | 提交评审 | `reviewing` |
| `reviewing` | 评审通过 | `approved` |
| `reviewing` | 评审驳回 | `rejected` |

### 非法流转

- `approved` 不允许再变更为 `rejected`。
- `rejected` 首期不支持直接重开，需复制新单据。

## 6. 薪资状态机

### 状态集合

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿 |
| `confirmed` | 已确认 |
| `archived` | 已归档 |

### 合法流转

| 当前状态 | 动作 | 下一状态 |
| --- | --- | --- |
| `draft` | 确认薪资 | `confirmed` |
| `confirmed` | 归档 | `archived` |

### 非法流转

- `archived` 不允许再修改金额。
- `confirmed` 后若要调整，只允许通过调整记录，不直接覆盖原记录。

## 非法流转处理方式

1. 后端返回业务失败，`message` 明确指出原因。
2. 前端显示错误提示并保持当前页面状态不变。
3. 不允许因为无效动作而“假成功”刷新列表。
