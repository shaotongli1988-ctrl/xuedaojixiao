# 2026-04-19 导航信息架构收敛设计

## 背景

当前顶部一级入口过多，用户在主工作流之间切换成本高。目标不是重做业务，而是在尽量不改现有路由、权限键和页面文件的前提下，先完成导航层级收敛。

本次讨论后的唯一目标是：

1. 将顶部一级入口压缩到稳定的少量业务域。
2. 保留现有主流程可达性，不把高频页面埋深。
3. 优先做菜单归组和命名调整，暂不扩大到业务重构。

## 决策结论

一级入口固定为 5 个：

1. 绩效中心
2. 人才中心
3. 目标&计划
4. 行政协同
5. 采购&资产

其中：

1. `人才中心 = 招聘中心 + 培训中心`
2. `行政协同 = 原行政协同 + 会议管理 + 合同管理`
3. `采购&资产 = 采购管理 + 资产管理`
4. 原一级 `班主任化` 取消，改为 `目标&计划` 下的二级菜单

## 设计原则

### 做什么

1. 只调整一级入口名称、一级归属和二级承载关系。
2. 为 `目标&计划` 提供一个聚合承载页 `目标&计划总览`。
3. 把 `班主任化` 作为 `目标&计划` 下的二级菜单保留。
4. `班主任化` 下暂时继续保留 4 个现有子页，不合成单页工作台。

### 不做什么

1. 不在本阶段统一改写现有页面路由。
2. 不在本阶段统一改写权限键。
3. 不在本阶段重构现有页面内部逻辑。
4. 不在本阶段合并状态机、接口契约或数据模型。
5. 不在本阶段把 `班主任化` 4 个页面收成一个超级单页。

## 目标结构

### 1. 绩效中心

保留绩效主链，不再承接班主任化。

建议二级菜单：

1. 绩效驾驶舱
2. 我的考核
3. 已发起考核
4. 待我审批
5. 指标库
6. 360环评
7. 建议管理
8. PIP 追踪
9. 晋升管理
10. 薪资管理

### 2. 人才中心

由招聘和培训合并而来，先合并导航，不先合并业务模型。

建议二级菜单：

1. 招聘中心
2. 招聘计划
3. 职位标准
4. 招聘简历池
5. 招聘面试
6. 人才资产
7. 录用管理
8. 培训课程管理
9. 课程学习与考试
10. 能力地图
11. 证书管理

说明：

1. 保留招聘统一入口 `recruitment-center`。
2. 培训相关页面先并入同一一级业务域，不要求立刻改成统一首页。

### 3. 目标&计划

这是新的一级业务域，用来承接目标类聚合入口和班主任化二级目录。

建议二级菜单：

1. 目标&计划总览
2. 班主任化

其中：

1. `目标&计划总览` 先承接 `目标地图`，首期允许直接复用现有 `/performance/goals` 作为总览路由。
2. 后续如果新增其他目标/计划类聚合内容，继续优先进入总览页，不再平铺多个一级入口。
3. `班主任化` 作为二级目录继续保留以下子页：
   1. 首页看板
   2. 班主任列表
   3. 我的待跟进
   4. 班级列表

### 4. 行政协同

由原行政协同、会议管理、合同管理合并而来。

建议二级菜单：

1. 年检材料
2. 荣誉管理
3. 宣传资料
4. 美工协同
5. 快递协同
6. 文件管理
7. 知识库
8. 车辆管理
9. 知识产权管理
10. 会议管理
11. 合同管理

说明：

1. 本阶段只改挂载归属，不把会议、合同强行融进行政协同页面壳。
2. 若后续导航再收敛，可考虑补一个行政协同首页，但不作为本阶段前置条件。

### 5. 采购&资产

由采购管理和资产管理合并而来，但先保持采购和资产的内部流程边界。

建议左栏核心入口：

1. 采购工作台
2. 供应商管理
3. 采购转资产
4. 资产台账
5. 资产作业
6. 资产分析

其中：

1. `采购工作台` 折叠承接：
   - 采购订单
   - 询价管理
   - 采购审批
   - 订单管理
   - 收货管理
   - 采购报表
2. `采购转资产` 复用现有 `/performance/asset/procurement`，只改导航命名，不改页面事实源。
3. `资产作业` 折叠承接：
   - 领用归还
   - 维护保养
   - 资产调拨
   - 资产盘点
   - 报废管理
4. `资产分析` 折叠承接：
   - 资产总览（复用现有 `资产首页` 路由 `/performance/asset/dashboard`）
   - 资产报表
   - 折旧管理
5. `资产分析` 下的 `本日 / 本周 / 本月动作总览` 已并入 `资产总览` 内容区，不新增独立左栏核心入口。

说明：

1. 采购与资产先共用一级入口，不要求合并成一套状态流。
2. `采购转资产` 是采购链进入资产链的唯一显式衔接点。
3. `资产台账` 仍是资产侧后续作业的主事实源，不因为左栏收口而改变。
4. 后续如需增强可辨识性，可在二级层补“采购”和“资产”视觉分区，但不新增第 7 个核心入口。

## 实施策略

推荐分两步实施：

### 第一步：只改导航承载

1. 调整前端 `performanceDomainGroups` 一级业务域分组。
2. 调整文档中的一级业务域归属说明。
3. 增加 `目标&计划` 聚合入口定义。
4. 将 `班主任化` 从一级组下沉为 `目标&计划` 的二级目录。

### 第二步：补聚合页

1. 首期优先复用现有 `/performance/goals` 作为 `目标&计划总览` 的实际落点。
2. 如果后续确认需要真正的聚合首页，再新增独立页面承载。
3. 不要求一步到位吞并所有目标类页面。

## 路由与兼容性策略

本设计明确要求兼容优先：

1. 现有页面路由默认保持不变。
2. 现有权限键默认保持不变。
3. 现有页面文件路径默认保持不变。
4. 菜单归组变化不应改变页面的唯一业务事实源。

这意味着：

1. 本次首先是导航 IA 调整，不是接口或权限体系改造。
2. 若后续需要路由别名或重定向，应单独评估，不与本轮导航收敛绑定。

## 风险与约束

### 风险

1. `人才中心` 会比现有单一模块更胖，需要后续考虑是否补人才首页。
2. `行政协同` 命名语义会变宽，需要在文档和视觉上解释其承载范围。
3. `采购&资产` 下菜单较多，二级可读性依赖清晰分区。
4. `目标&计划总览` 如果持续加内容，后续可能演变成过重聚合页。

### 约束

1. 不允许因为收入口而顺手改权限、接口或表结构。
2. 不允许把未确认的业务归属硬塞进 `目标&计划总览`。
3. 不允许在没有验证用户习惯前，直接把 `班主任化` 四页收成单页工作台。

## 验收标准

### 信息架构验收

1. 顶部一级入口减少为 5 个。
2. 原一级 `班主任化` 不再单独出现。
3. `目标&计划` 成为新的一级入口。
4. `班主任化` 在 `目标&计划` 下以二级目录形式出现。
5. `目标地图` 被纳入 `目标&计划总览` 承载范围。
6. `采购&资产` 左栏核心入口收敛为 6 个。
7. 原采购链和资产链页面仍可通过折叠组到达。

### 兼容性验收

1. 现有高频页面仍可通过菜单到达。
2. 现有页面路由不因菜单归组调整而失效。
3. 现有权限控制不因菜单归组调整而漂移。
4. 现有 `PurchaseWorkspace` 与资产各页面壳继续复用，不强行合并成新超级页。

### 实施边界验收

1. 若本轮只完成菜单归组，则不应被视为聚合页未做完的缺陷。
2. `目标&计划总览` 若进入实现，应有清晰范围，不得一次性吞并所有相关页面。
