这份文档只记录 **当前项目自己的具体坑、历史兼容点、项目偏差和踩坑结论**。

不要把所有通用规则都堆在这里。

请配合基座机制文档一起使用：

- `docs/00-system/项目专属坑位文档维护机制.md`

## 1. 使用说明

开发时请分两层：

### 第一层：先看通用规则

先看：

- 项目根 `AGENTS.md`
- 当前端 / 后端目录下的 `AGENTS.md`
- 对应数据库 / 技术栈的通用规范

### 第二层：再看项目专属坑

再看本文件。

这里记录的是：

- 当前项目真实踩过的坑
- 当前项目与 base 的偏差
- 当前项目当前有效的规避方式

## 4. 维护规则

以下情况出现时，建议更新本文件：

- 用户明确说“把这个坑记录一下”
- 已经发生过一次真实踩坑，并且根因明确
- 这是当前项目特有的问题，而不是所有项目的共性规则
- 如果不记录，后续很可能再次踩到

如果后来发现这不是项目私有坑，而是技术栈共性问题：

- 再考虑上提到基座规范
