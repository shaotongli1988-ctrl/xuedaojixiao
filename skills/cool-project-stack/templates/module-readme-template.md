# 多期模块 README 模板

## 这份 README 是干什么的

这份 README 是当前模块的长期入口文档。

它主要负责 5 件事：

1. 先说明这个模块为什么存在、要解决什么问题
2. 说明这个模块目前拆成了哪几期、后面准备怎么继续迭代
3. 告诉团队成员和 AI 第一次接手时，应该先看什么、后看什么
4. 明确当前正在开发、联调或维护的是哪一版
5. 把跨版本长期有效的约束说明、各版本文档入口统一串起来

一句话理解：

**如果一个功能会拆成多期做，这份 README 就是这个模块的总入口，不是可有可无的说明页。**

---

## 一句话提醒

**多期功能不要把各期文档散落在 `docs/` 根目录或多个无关目录里，统一放在同一个模块目录下，并由这份 README 串起来。**

## 当前推荐开发顺序

> 这部分写“现在团队建议先做什么，再做什么”，不是写所有历史过程。

当前推荐按下面顺序推进：

1. 先确认当前版本目标与边界
2. 再确认当前版本继承上一版哪些规则
3. 再补当前版本的需求 / 设计 / 实施文档
4. 再按实施顺序开发与联调
5. 开发完成后补测试清单、交付说明，并回写本 README

如果当前版本基于旧版本增强，开始前先明确：

- 继承上一版哪些内容
- 本版新增哪些能力
- 本版调整哪些规则
- 本版废弃哪些旧做法

## 维护约定

1. 只要模块继续迭代，这份 README 就要持续更新
2. 新增版本时，先更新本 README，再补该版本文档
3. 新版本如果基于旧版本增强，要写清“基于哪一版、改了什么、哪些继续沿用”
4. 跨版本长期有效的内容，优先沉淀到模块级长期文档，不要在每个版本重复抄写
5. 如果当前推荐阅读顺序变了，也要同步更新本 README
