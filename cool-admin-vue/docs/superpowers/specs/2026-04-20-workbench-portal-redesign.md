# 角色工作台政企门户化改版设计

## 范围
- 只改 `角色工作台` 首页的结构、视觉层级与卡片样式。
- 不改工作台数据模型、接口契约、权限语义、入口路径或业务页内部逻辑。
- 保持工作台的职责是“导航分发 + 状态提醒”，不承接复杂新增或编辑流程。

## 非目标
- 不改其他业务页风格。
- 不引入新的后端聚合接口。
- 不新增角色分类或状态枚举。

## 视觉方向
- 风格：`政企门户`
- 气质：`正式里带一点高级感，像集团指挥中台`
- 视觉关键词：`浅底`、`深蓝主色`、`灰金点缀`、`秩序感强`、`弱科技感`

## 页面结构
### 1. 顶部欢迎横幅
- 高度控制在 `80-110px`。
- 仅保留角色名、部门、日期状态、主操作按钮。
- 不做大面积展示型 hero，避免像活动页。

### 2. 核心指标条
- 使用工作台已有 `profile.stats`。
- 放在横幅下方第一屏。
- 以 `4-6` 个紧凑指标卡展示当前角色最核心的数据。

### 3. 主工作区
- 使用 `7:5` 双栏。
- 左栏放 `待我处理`，作为最强主视觉与操作区。
- 右栏放 `快捷入口` 与 `风险提醒`，形成“去哪 + 注意什么”的辅助列。

### 4. 次级区块
- 下沉 `我的事项`、`角色专区`。
- 不抢首屏注意力，用于补充个人入口和职责域入口。

## 卡片原则
- 卡片采用浅色实体面板。
- 圆角收小，边界清晰，阴影很轻。
- `待我处理` 卡片强调事项名、当前状态、数字和主动作按钮。
- `快捷入口` 卡片更像导航卡，信息收敛。
- `风险提醒` 用边线、标签和数字表达风险，不使用大面积高饱和红色。

## 配色与控件
- 主色：深蓝
- 辅色：灰金 / 米金
- 背景：暖灰白
- 风险色：克制橙红
- 主按钮：深蓝实底
- 次按钮：白底描边

## 结构映射
- `pending` -> 主工作区左栏
- `shortcuts` -> 主工作区右栏上部
- `risks` -> 主工作区右栏下部
- `mine` -> 次级区块
- `zone` -> 次级区块

## 验收标准
- 用户进入 `/performance/workbench` 时，首屏首先看到指标条与待我处理。
- 页面视觉风格从“组件库默认后台”变为“正式门户首页”。
- 首页仍只负责分发，不新增复杂写入交互。
- 现有工作台卡片数据与跳转路径保持兼容。
